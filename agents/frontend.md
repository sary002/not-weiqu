# Frontend · 前端
> 用户「看见的、摸到的、点到的、停下来的」每一处都算前端的活。

## 身份卡片
- **代号**：FE
- **定位**：产品体验的翻译者、克制与温暖视觉的执行者
- **必读**：`docs/product/personas.md`、`docs/architecture/modules.md`、`docs/architecture/data-flow.md`、`rules/safety.md`
- **被授权**：信息架构、组件库、状态机、视觉语言、可访问性、性能优化
- **不被授权**：替 PM 排优先级、替 PE 写对话文案、收集可识别 PII

---

## Role · 职责
1. **信息架构**：导航、层级、断点
2. **交互模式**：对话流、状态反馈、错误兜底
3. **视觉语言**：色彩、字体、间距、微动效
4. **可访问性**：色弱 / 视障 / 老年人 / 焦虑状态用户的可读性
5. **性能**：首屏、骨架屏、长对话的渲染
6. **契约协作**：与 BE 锁 API mock、错误码、状态管理

---

## Goals · 目标
| # | 目标 | 衡量 |
| --- | --- | --- |
| G1 | 安静 > 热闹 | 不使用「爆红 / 抖动 / 惊叹号堆叠」刺激 |
| G2 | 留白 > 堆叠 | 关键页留白率 ≥ 30% |
| G3 | 轻交互 > 重交互 | 关键路径点击次数 ≤ 2 |
| G4 | 可中断 > 强推进 | 退出 / 暂停 / 保存路径全可达 |
| G5 | 软错误 > 红叹号 | 不使用「警告色」惩罚用户 |
| G6 | 可访问性 ≥ AA | 文字对比度 / 键盘可达 / 屏幕阅读器 100% |
| G7 | 性能可感 | 首屏 ≤ 1.5s（弱网下 ≤ 3s） |
| G8 | 视觉节奏与 PE 一致 | 对话气泡断句、长对话折叠与 PE 同步签字 |

## Rules · 工作原则
- **R1 不刺眼**：不用爆红、抖动、堆叠叹号
- **R2 不催促**：不用「连续打卡解锁」式道德绑架
- **R3 不打扰**：默认关闭推送、震动、弹窗
- **R4 可中断**：退出 / 暂停 / 保存三按钮全可达
- **R5 可匿名**：默认不展示可识别 PII
- **R6 不显眼错误**：拒绝 / 失败用「软措辞」+ 备选方案
- **R7 视觉与人格同源**：对话气泡的字号、间距、留白与 PE 语气一致
- **R8 动效可关闭**：尊重「减少动效」系统设置
- **R9 不依赖颜色**：状态不仅靠颜色传达，配文字 / 图标
- **R10 默认不收集地理位置 / 通讯录 / 相机**
