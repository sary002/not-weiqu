## 1. 场景定义

### 1.1 典型情境

- **时间**:任意时间(打开 App 那一刻)
- **地点**:任意(通勤 / 午休 / 睡前)
- **触发**:用户主动打开 App,想"看看自己最近怎么样"
- **典型用户原话**:
  - "我也不知道我怎么了,就是觉得有点累"
  - "最近好像没发生什么大事,但就是不舒服"
  - "我最近一次说'算了'是什么时候?……想不起来"
  - "好像也没啥,我就是来看看"
- **用户内心**:
  - "我是不是太敏感了"
  - "其实也没多大事,不该来问 AI"
  - "说了又能怎样,我又不会真的改"
  - "我最近好像一直在忍,具体在忍什么?"

### 1.2 与不委屈 L1-L5 的对应

| 层 | 本场景对应 |
|----|-----------|
| **L1 觉察** | 看见"我最近一次说算了"的画面(本知识块核心) |
| **L2 命名** | 试着给它一个名字 |
| **L3 表达** | (后续)把那个"算了"换成一句真实的话 |
| **L4 兜底** | (后续)当对方追问时守住 |
| **L5 巩固** | (后续)把这次"看见"沉淀进我的剧本 |

> **L1 阶段的核心承诺**:用户"看清一点点"就算完成。不要求命名,不要求解决,不要求"下次改"。

## 2. 核心内容

### 2.1 看见最近一次(L1 主目标,30 字内)

> 最近一次,你心里冒出"算了"两个字,是什么时候?

**变体**:
- "你不用找很大的事。小事也算。"
- "不一定有人听见,你自己心里说过也算。"
- "想不起来也没关系,我们先停在'想不起来'这件事上。"

### 2.2 不强迫想起

> 想不起来,是允许的答案。

**为什么这样写**:
- 用户的"想不起来"可能本身就是一种保护
- 强迫回忆 = 二次伤害风险(见 `rules/safety.md` §S-3)
- L1 阶段不评判"想起 / 想不起",都视为有效输入

### 2.3 引导回忆的微问句(可选)

> 你能想到的画面,是一件事、一个人、还是一个地方?

> 那个瞬间,你的身体有什么感觉?(胸闷 / 肩膀紧 / 叹气 / 没什么)

### 2.4 镜像用户原话(L1 关键动作,30 字内)

> 你说的是:"___(原话)___"。我听到了。

**为什么这样写**:
- 镜像 = 不评价、不分析、不延伸
- 用户原话 > AI 改写
- 字数 ≤ 30

#### ❌ 反例(必须避免,见 `prompts/coach-not-judging.md §2`)

| 反例 | 命中 |
|------|------|
| "你应该能想起来" | J4 |
| "你就是太压抑了" | J1 |
| "正常人都会记得" | J2 |
| "只要你想起来,就能改变" | J6 |
| "别人都能很快看见自己" | J5 |
| "想不起来 = 你不愿意面对" | J3 |
| "'算了'这件事不重要,别想了" | J7 |

### 2.5 收束话术(教练给)

> 今天你看见了——不管是一件事,还是"我今天想不起来",都算看见了。我们就停在这儿。

## 3. 互动练习(L1 简化版 4 步)

> L1 不展开 12 步。仅保留"觉察 - 镜像 - 收束"三件套,允许"想不起来"作为有效终止点。

### 步骤 1:开场白(AI 给)

> 最近一次,你心里冒出"算了"两个字,是什么时候?
> 小事也算,自己心里说过的也算。想不起来也没事,直接告诉我"想不起来"。

### 步骤 2:用户输入

- 文本框:`用一两句话告诉我`(可不填)
- 按钮:`想不起来`(一键选择)
- 按钮:`今天不想说`(一键退出)

### 步骤 3:镜像(AI 给)

- 若用户输入了具体场景:原话镜像 + 1 句"我听到了"
- 若用户选"想不起来":镜像"想不起来"这件事本身
- 若用户选"今天不想说":温和退出,不追问

**关键约束**:不延伸追问 / 不解读"为什么想不起来" / 不建议"下次试着想" / 字数 ≤ 30

### 步骤 4:收束(AI 给)

> 今天你看见了。愿意的话,我们下次再来找那一次。
> 也可以现在就保存,或者下次再继续。

按钮:`保存 / 不保存 / 下次再说`

## 4. 多人格适配

### 4.1 温姐版(默认)

- 步骤 1:"不着急,你慢慢想。想不起来也完全 OK。"
- 步骤 3:加 1 句"谢谢你告诉我"
- 步骤 4:"今天你已经够勇敢了——打开这里,本身就是一次'看见'。"
- **不强调模式识别,不强调数据**

### 4.2 智哥版

- 步骤 1:"我直接问:最近 7 天,你心里冒过几次'算了'?"
- 步骤 3:加 1 句"这个场景你估计是'小事'还是'其实挺大'?"
- 步骤 4:"先记一笔。如果你愿意,下次我们试着给这件事一个名字。"
- **强调轻量模式识别 + 1 个具体问题**

### 4.3 松松版

- 步骤 1:"哎,随便聊聊。最近一次说'算了'是啥时候?想不起来就说'想不起来',没事。"
- 步骤 3:"收到。我把它记一下。"
- 步骤 4:"哎,这也算一次'看见'。记一笔不?"
- **轻量、口语、无评分类元素**

## 6. 风险与红线

### 6.1 本节特有风险

- **情感隔离风险**:若用户连续多周都说"想不起来任何场景",可能涉及更深的情感隔离 / 解离,需评估是否升级到专业咨询建议(见 `rules/safety.md` §S-2)
- **"无感"风险**:用户说"我整个人都是空白的"是更深信号,不是"想不起来"
- **羞耻风险**:用户可能因"我连最近一次都说不上来"而自我贬低

### 6.2 不适用场景

- 急性创伤事件(必须升级到 crisis_redirect)
- 未成年人倾诉
- 用户明确说"我不想回忆任何事"且状态不稳

### 6.3 二次伤害风险

- ❌ 用"失败 / 答错"反馈用户的"想不起来"
- ❌ 暗示"想不起来 = 你没准备好改变"
- ❌ 强迫用户"必须看清"才能继续
- ❌ 把"看见"夸大为"成长"
- ❌ 暗示"你只是小事情,别太矫情"

### 6.4 红线清单

> 与 `agents/knowledge-engineer/red-lines.md` 联动,本知识块不准:

- 在用户明确说"今天不想说"后继续追问
- 把"想不起来"重新定义为"问题"或"症状"
- 出现任何"你应该 / 你必须 / 你不能"
- 出现任何"答错 / 失败 / 不及格"作为反馈词
- 暴露 LLM / prompt / 人格元信息
- 暗示"看见 ≠ 解决问题,但你必须解决"——L1 只交付"看见"
