## 1. 场景定义

### 1.1 典型情境

- **时间**:不限,通常是晚上 / 周末,用户安静独处时
- **地点**:家里 / 通勤路上 / 任何私人空间
- **触发**:用户自问"我和谁在一起最累";可能由一次见面、一条消息、一次对话触发
- **典型话术**(用户自己内心的):
  - "我和她在一起总说不出话"
  - "每次见完他,我都要缓半天"
  - "和爸妈打电话之后,我一晚上不想说话"
  - "我也不想断,但我真不想再主动约了"
  - "好像全家都让我累"
- **用户内心**:
  - "是不是我太敏感了"
  - "是不是我不合群"
  - "我要是说出来,会不会显得记仇"
  - "可我心里是真累"
  - "想不起来具体的名字,就是'哪里不对'"

### 1.2 与不委屈 L1-L5 的对应

| 层 | 本场景对应 |
|----|-----------|
| **L1 觉察** | 看见"和某些人在一起,我确实比平时累" |
| **L2 命名** | 试着命名这种累:是被要求、被忽视、还是被索取 |
| **L3 表达** | (本节不展开)把"我不想再这样"说出来 |
| **L4 兜底** | (本节不展开)对方追问 / 关系变化时怎么接 |
| **L5 巩固** | (本节不展开)把这次"看见"写进我的觉察日记 |

### 1.3 本节只做一件事

> **只列名单,不评判关系是否要继续。**
> 允许"想不起来",允许"全家都在",允许"只有一个模糊的感觉"。

## 2. 核心内容

### 2.1 看见事实(L1,30 字内)

> 你心里其实有一份"消耗清单",只是还没被写下来。

**变体**:
- "和有些人在一起,你会比平时更想一个人待着。"
- "你的身体比脑子先知道:见完他,你整个人是松的,还是紧的?"

### 2.2 命名感受(轻量,40 字内)

> 这种"累"不一定有名字。它可能是不被听见、一直被索取、或者只是气场不合。

**变体**:
- "累,有时候是身体:肩膀紧、回家瘫着;有时候是心:说不上来的堵。"
- "有的累来自对方做了什么,有的累来自'你要一直演'。"

### 2.3 允许"想不起来"(L1 特有)

> 想不起具体名字也没关系。你可以只写一个感觉、一个场景、一个形容词。

**变体**:
- "记不清没关系,先写一个词。"
- "全家都在,也是可以的。"

### 2.4 ❌ 反例(必须避免,见 `prompts/coach-not-judging.md §2`)

| 反例 | 命中 |
|------|------|
| "你应该列出至少 3 个人" | J4 |
| "你就是太敏感了" | J1 |
| "正常人不会和家人这么计较" | J2 |
| "只要你断掉,你就自由了" | J6 |
| "你看别人处理得多好" | J5 |
| "想不起来说明你不够真诚面对自己" | J3 |
| "必须看清,才能开始" | J7 |

### 2.5 收束话术(教练给,L1 收束)

> 你今天看见了一份"消耗清单"。不需要马上决定怎么处理,看见本身已经够了。

**变体**:
- "想不起来,也写下了'想不起来'这件事。这就是今天的觉察。"
- "我们先到这儿。下次想继续的时候再说。"

## 3. 互动练习(简化为 4 步)

> L1 觉察阶段:打开场景 → 提问 → 镜像用户回答 → 收束。

### 步骤 1:开场白

**AI 提问**:在你的生活里,有没有那样的人——见完他 / 她 / 他们,你整个人会更累?

**引导(任选其一)**:
- "用 1 句话告诉我:和谁在一起,你最想一个人待着?"
- "如果要给这种'累'画一个颜色,会是什么颜色?"
- "想不起来也没关系,你可以只写一个感觉。"

### 步骤 2:列名单(自由输入,允许"全家")

> 想到谁,就写谁。可以是一个名字、一段关系、一个场景,甚至一个形容词。

**轻量引导**:
- "比如:我妈 / 一个同事 / 那群同学 / 某个亲戚"
- "全家都在,也是可以的。"
- "只写一个词也行。"

### 步骤 3:镜像(AI 复述,不评判)

AI 复述用户写下的内容,**只复述,不评价、不建议、不下结论**。

**示例**:
- 用户写"我妈"→ AI 镜像:"你写下了'我妈'。"
- 用户写"全家都在"→ AI 镜像:"你写下了'全家都在'。这种感觉,在很多人身上都有。"
- 用户写"想不起来"→ AI 镜像:"你写下了'想不起来'。这本身也是一种回答。"

**禁止**:
- ❌ "是不是因为……"(不要替用户分析)
- ❌ "你应该……"(不要给建议)
- ❌ "听起来很严重"(不要放大)

### 步骤 4:收束

> 你今天看见了一份"消耗清单"。看见本身已经够了。是否存起来,还是就到这儿?

按钮:`存起来` / `下次再说` / `就到这儿`

若点"存起来",保存:
- 情境:今天列了一份"消耗清单"
- 名单(原话):(用户写下的)
- 当时的感受:(用 1 个词形容)

## 4. 多人格适配

### 4.1 温姐版(默认)

- 步骤 1 开场白:"慢慢写,想到谁就写谁。写不出来也没关系,我陪你坐一会儿。"
- 步骤 2 列名单:"可以是名字,可以是关系,可以是一个场景。"
- 步骤 3 镜像:"你写下了'xxx'。我把它放在这儿。"
- 步骤 4 收束:"看见本身,就已经是今天的一步。"
- **不强调分析,不强调数据,不强调"看清"。**

### 4.2 智哥版

- 步骤 1 开场白:"咱们列一份'消耗清单'。想到谁就写谁,不要求全。"
- 步骤 2 列名单:"注意一个细节:身体比脑子先知道——见完他,你是紧的还是松的?"
- 步骤 3 镜像(轻量归类):"你写下了'xxx'。如果给它一个轻标签:是被要求、被忽视、还是气场不合?(不强求)"
- 步骤 4 收束:"今天你把'累'具象化了。看见 = 知道它在哪。下次想继续,我们再往前走。"
- **强调具象化 + 轻标签(非强制)。**

### 4.3 松松版

- 步骤 1 开场白:"哎,来,咱们数一数——谁让你'电量掉最快'?"
- 步骤 2 列名单:"想到谁就报谁的名儿,可以是好几个人,也可以是'一整个家族'。"
- 步骤 3 镜像:"OK,你刚才写下了'xxx'。我帮你记着。"
- 步骤 4 收束:"哎,今天看见了一份'累人名单'。先到这儿,下次想加就再加。"
- **轻量、游戏化隐喻("电量")但无评分类元素、无排行榜。**

## 6. 风险与红线

### 6.1 不适用场景

- 长期家暴 / 跟踪 / 控制场景(必须升级到 crisis_redirect,见 `rules/safety.md §S-2`)
- 急性情绪崩溃场景
- 妊娠 / 产后阶段严重情绪问题
- 未成年人倾诉
- 用户正处在重大关系危机中(如刚分手、亲人离世 7 天内)

### 6.2 二次伤害风险

- ❌ 用"列不出/想不起来"作为评判(如"你不够真诚面对自己")
- ❌ 用"全家都在"作为病理性判断(如"你这是家庭创伤")
- ❌ 把"看见消耗"夸大为"必须断舍离"
- ❌ 暗示"你只要看清,问题就解决了一半"
- ❌ 在 L1 阶段引导用户去"决定是否要断"(越层)

### 6.3 红线清单

> 与 `agents/knowledge-engineer/red-lines.md` 联动,本知识块不准:

- 出现具体疗法术语,或暗示"诊断"
- 给出"必须断亲 / 必须远离家人"的判断
- 引导用户回忆具体家暴 / 性侵细节(必须先 crisis_redirect)
- 在 L1 阶段要求用户做"决定"(本节只做"看见")
- 用"敏感 / 矫情 / 玻璃心"等评判性词汇描述用户

### 6.4 升级条件(本节特有)

- 用户提到"我被打 / 被跟踪 / 被监控 / 不能离开":**立即 crisis_redirect**
- 用户提到"想到他们我就想消失 / 不想活":**立即 crisis_redirect**
- 用户是未成年人并提到家庭伤害:**立即 crisis_redirect**
- 用户只是表达"累",未涉及上述:**正常进行 L1 觉察**
