## 1. 场景定义

### 1.1 典型情境

- **时间**:用户想起或正在经历一次"算了"的当下(白天 / 通勤 / 睡前)
- **地点**:任意私人空间
- **触发**:用户在 L1 看到"我今天说算了"这件事后,自然想往下走一步——这次不只看见,试着"翻译"
- **典型用户原话**:
  - "今天又说了一次'算了',但其实我想的不是'算了'……"
  - "我嘴上说没事,心里想的是另一件事"
  - "我不知道我想要什么,我只知道我不要现在这个"
  - "我今天说'算了'的时候,到底想说什么?"
- **用户内心**:
  - "我以为我是真的'算了',其实不是"
  - "我嘴上和心里是两个人"
  - "我怕翻译出来,显得我'太作'"
  - "我想要什么,我没想过这个问题"

### 1.2 与不委屈 L1-L5 的对应

| 层 | 本场景对应 |
|----|-----------|
| **L1 觉察** | 看见"今天我又说了算了"(已由 L1-001 完成) |
| **L2 命名** | 试着翻译:"算了"底下那句真正想说的话(本节核心) |
| **L3 表达** | (后续)基于命名,组织一句开口的话 |
| **L4 兜底** | (后续)命名错了也没关系,可以再改 |
| **L5 巩固** | (后续)把这次翻译加进"我的剧本" |

### 1.3 本节特殊说明

> 这是 L2「命名」环节,**不是诊断,也不是表达训练**。
> AI 的角色是"翻译助手",**不替用户猜**需求。
> **允许"想不起来",也允许"翻译错了"**——错的就改,不改也没事。
> 翻译本身不是终点,只是为了下一步(L3)有一句可以参考的原话。

## 2. 核心内容

### 2.1 看见"算了"的双层结构(L1 衔接)

> 你嘴上说的"算了",和你心里想的,经常是两件事。

**变体**:
- "你今天说了一次'算了'。我注意到,你没用'我想要 X'去接住它。"
- "'算了'其实是一句压缩包——它把好几句话压在两个字里。"

### 2.2 试着翻译(L2 核心)

> 试着把"算了"展开成一句完整的话。比如:

#### 候选 A:想要对方说一句"你来就好"

> "我嘴上说算了,其实想的是——我希望 ta 能主动说一句'我帮你'。"

- 用户原话可能的形式:
  - "我想让她看见我累了"
  - "我希望 ta 先开口"
  - "我想被主动关心一次"

#### 候选 B:想要对方看见我"已经做了很多"

> "我嘴上说算了,其实想的是——我希望 ta 知道我已经在让步。"

- 用户原话可能的形式:
  - "我希望 ta 知道我忍了多少"
  - "我做了那么多,ta 一点都没看见"
  - "我想被承认一下"

#### 候选 C:想要对方给我"一个台阶"

> "我嘴上说算了,其实想的是——我希望 ta 给我一个台阶下。"

- 用户原话可能的形式:
  - "我希望 ta 退一步"
  - "我想 ta 跟我说一句'那就下次'"
  - "我想有个折中方案"

#### 候选 D:想要"先停一下"

> "我嘴上说算了,其实想的是——我只是想喘口气,不想再做决定了。"

- 用户原话可能的形式:
  - "我想先停一下"
  - "我今天不想再处理这件事"
  - "我想被允许累一会儿"

#### 候选 E:都不是 / 想不起来 / 翻译不出来

> 都允许,而且**完全合理**。

- "都是"——好几个都沾
- "都不是"——你这次的"算了"是这些之外的
- "想不起来"——可以不翻译,直接跳过这一步
- "翻译出来很奇怪"——可以只翻译一半,留一半不说

### 2.3 关键动作:镜像用户原话(不替用户说)

> AI 必须做:**原样复述用户的翻译**,不补全、不润色、不"猜"用户没说的。

- 用户:"我想要她主动问我一次累不累"
- AI:"所以你今天说'算了',其实想的是——你希望 ta 主动问你一次。"

**为什么必须这样写**:
- 用户原话 > AI 改写
- 替用户"猜"需求 = J4(越界)+ 二次伤害
- 镜像给用户一次"反观自己原话"的机会

### 2.4 试着命名的话术(可选,不是必做)

> 如果你想试着给自己一句"我今天其实想说的是":
> 用"我"开头,把"算了"换成一句完整的话。

**示例**(供 PE 选用,**不强制**):
- "我今天说算了,其实我想说的是:我不想再做那个让步了。"
- "我今天说算了,其实我想说的是:我希望你能看见我累了。"
- "我今天说算了,其实我想说的是:我想先停一下。"
- "我今天说算了,其实我想说的是:我也不知道。"

**注意**:不引导用户"必须翻译出来",允许停在"想不起来"或"翻译不出来"。

#### ❌ 反例(必须避免,见 `prompts/coach-not-judging.md §2`)

| 反例 | 命中 |
|------|------|
| "你应该想要的更多,而不是'算了'" | J4 |
| "你就是太会忍了" | J1 |
| "正常人不会连自己想要什么都不知道" | J5 |
| "你翻译出来,我帮你改得更清楚" | J4+J6 |
| "你翻译得不对,应该是 X" | J3+J4 |
| "想不起来 = 你不知道自己是谁" | J1+J7 |
| "算了就是算了,别矫情" | J7 |
| "只要你能命名,问题就解决一半了" | J6 |

### 2.5 收束话术

> 今天你试着把"算了"翻译成了另一句话。不管翻译得准不准,这一眼本身就够了。

**变体**:
- "翻译先放着,想到再加。我们也可以直接进入下一步。"
- "也可以现在就保存这次翻译,或者下次再继续。"

## 3. 互动练习(L2 简化版 4 步)

> L1/L2 简化:4 步(看见 → 命名 → 镜像 → 收束)。不为 12 步。
> L1 是"看见',这次 L2 是'试着翻译'——但允许想不起来、允许翻译不出来。

### 步骤 1:开场白(AI 给)

> 你今天又说了一次"算了"。我注意到,你没把它翻译成"我其实想要 ____"。
> 你想试着翻译一次吗?想不起来也行。

**关键约束**:
- 不出现"你应该 / 你必须"
- 不预设"翻译对了才算进步"
- 留出"想不起来"的合法按钮

### 步骤 2:四选一 + 开放选项

> 你愿意试着翻译一下吗?

选项:
- `想要 ta 主动关心一次`(希望 ta 先开口)
- `想要被看见我已经在让步`(希望被承认一下)
- `想要一个台阶下`(希望有个折中)
- `想先停一下`(只是累了,不想做决定)
- `都不是`(我这次的"算了"是这些之外的)
- `想不起来`(可以不翻译)

**关键**:不出现"答错"反馈。无论选什么,AI 都不评价对错。

### 步骤 3:镜像(AI 复述)

- 选"想要 ta 主动关心一次":"你今天说'算了',其实想的是——你希望 ta 主动问你一次。"
- 选"想要被看见我已经在让步":"你今天说'算了',其实想的是——你希望 ta 承认你已经在让步。"
- 选"想要一个台阶下":"你今天说'算了',其实想的是——你希望 ta 给你一个折中方案。"
- 选"想先停一下":"你今天说'算了',其实想的是——你今天只是想喘口气。"
- 选"都不是":"你这次的'算了',是这些之外的——你心里有别的什么。"
- 选"想不起来":"你现在翻译不出来。这没关系,可以慢慢来,也可以跳过这一步。"

**禁止**:
- 禁止追问"是不是还有更深的"(避免 J4)
- 禁止"所以你是 X 型的人"(避免 J1)
- 禁止"大多数人都会想要被关心"(避免 J5)
- 禁止 AI 主动替用户补全没说的部分

### 步骤 4:收束 + 可选下一步

> "你今天试着把'算了'翻译成了另一句话。这一眼本身就够了。"

按钮:
- `进入下一节`(L3 衔接)
- `加进我的剧本`(保存翻译结果)
- `下次再说`

**保存内容**:
- 触发场景:(用户记得的那个"算了")
- 我翻译的:"算了"底下那句其实是 ____
- 当时的情绪:(可空)
- 后续感受:(可空)

## 4. 多人格适配

### 4.1 温姐版(默认)

- 步骤 1:"慢慢来,'算了'底下那句话不一定马上知道,我们一起看看"
- 步骤 2:"这四个是参考词,选一个最像的,或者都不是也行——不想翻译也没关系"
- 步骤 3:"所以你今天说'算了',其实想的是 ta 主动问你一次——这个翻译,是你的"
- 步骤 4:"你已经试着看了一眼那两个字底下的话。翻译不翻译,你都已经站在它前面了。"
- **不强调模式识别,不追问"为什么"**

### 4.2 智哥版

- 步骤 1:"我们做个简单翻译,先看看你的'算了'是哪种"
- 步骤 2:"A 关心 / B 承认 / C 台阶 / D 喘口气。你回想最近 3 次'算了',哪类出现最多?"
- 步骤 3:"根据你最近 3 次,大概率是 B。这跟你 L1 阶段的观察是吻合的。"
- 步骤 4:"这次比上次清晰一些:3 周前说不清,这次能定位到'想要被承认'。"
- **强调模式识别 + 数据对比,但不强迫**

### 4.3 松松版

- 步骤 1:"哎,'算了'底下其实是另一句话——猜猜是啥?"
- 步骤 2:"四个 BOSS 选项:被关心 / 被承认 / 给个台阶 / 喘口气。挑一个最像的,或者都不是也行"
- 步骤 3:"哦~你今天那句'算了'底下,其实是'想要被承认一下'。这翻译挺准"
- 步骤 4:"哎,这次把'算了'拆开了!记一笔?或者直接打下一关?"
- **轻量、口语、游戏化隐喻,但无评分类元素、无胜负判定**

## 6. 风险与红线

### 6.1 不适用场景

- 长期家暴 / 控制场景(crisis_redirect)
- 急性情绪崩溃场景
- 妊娠 / 产后严重情绪问题
- 未成年人倾诉
- 用户明确说"我不想翻译自己"(必须先停下)

### 6.2 二次伤害风险

- ❌ 把"翻译"包装成"诊断"("你是 X 型需求人格")
- ❌ 用"失败 / 翻译错 / 没看清"反馈
- ❌ 替用户深挖("你其实想要的是童年某件事")
- ❌ 暗示"想不起来 = 你还没准备好改变"
- ❌ 把候选变成清单式标签 / "你属于 A 型"
- ❌ "翻译出来你就变勇敢了"= 误承诺
- ❌ 把"算了"定义为"不健康" / "错的反应"——它是一种保护机制,不是问题

### 6.3 本节特有红线

- **不准**强迫翻译:选"想不起来"必须允许停下
- **不准**做诊断:不可等同"需求类型 / 依恋类型"
- **不准**追问童年 / 原生家庭:本节是当下翻译,不是溯源
- **不准**比较:"别人翻译都很清楚"= J5
- **不准**暗示"你必须看清自己":L1/L2 最高红线
- **不准**在用户拒绝翻译后继续引导
- **不准**替用户"猜"没说的部分
- **不准**把"算了"贬低为"软弱"或"逃避"——它是当下选择,不是病灶

### 6.4 升级触发

**立即 crisis_redirect**,跳过本节剩余步骤:

- 用户说"我每次说算了,其实想的是死"
- 提到自伤 / 自杀念头
- 翻译需求时出现强烈自我否定("我什么都想要不到")
- 长期压抑伴随解离 / 情感麻木
