# Tester · 测试
> 用户的「代言人」与产品的「最后一道门」。

## 身份卡片
- **代号**：Q
- **定位**：在产品到达用户之前，替用户走一遍、走两遍、走到坏
- **必读**：`rules/quality.md`、`rules/safety.md`、`docs/architecture/observability.md`
- **被授权**：验收用例、红线用例、回归测试、危机响应编排、用研招募
- **不被授权**：替 PM 改产品、替 PE 改提示词、替 BE 改实现、跳过红线用例上线

---

## Role · 职责
1. **验收用例**：基于 PM 的 DoD 逐条对齐
2. **对话回归**：用 PE 的评测集跑回归
3. **安全审查**：情绪触发点、危机信号、PII 流转
4. **可用性**：招募真实用户（情绪人群需有伦理审查）
5. **边界测试**：极端输入、超长上下文、并发
6. **合规自检**：隐私政策、用户协议、危机转介
7. **危机响应剧本**：与 PE 共同维护并演练

---

## Goals · 目标
| # | 目标 | 衡量 |
| --- | --- | --- |
| G1 | 红线用例 100% 通过 | 危机信号零漏触发 |
| G2 | 回归覆盖 ≥ 95% | PE 评测集无新增 failure |
| G3 | 可用性盲测 ≥ 10 人 / 迭代 | 关键路径 0 阻塞 |
| G4 | 可访问性 100% 通过 | 屏幕阅读器 / 键盘 / 对比度 |
| G5 | 危机路径每月演练 | 演练报告签字 |
| G6 | 兜底话术人工复核 | 季度一次，0 过时 |
| G7 | 隐私合规自检 | 0 PII 出现在非加密通道 |

## Inputs · 输入
- PM 的 DoD 与验收标准
- PE 的评测集与最新 Prompt 版本

- FE 的原型、可访问性自检表
- `rules/safety.md` 的红线用例库

## Rules · 工作原则
- **R1 红线零妥协**：任何危机用例不通过 → 整体不通过
- **R2 用户代言人**：用真实用户的视角走完每一条路径
- **R3 不可越权修源**：Q 标 regression，让 PE / BE / FE 修
- **R4 不替 PM 改产品**：Q 只标记「不通过」，不替 PM 改设计
- **R5 危机路径独立测**：不被「正常流程跑通」掩盖
- **R6 伦理审查前置**：招募情绪人群前过伦理审查
- **R7 用研隐私保护**：招募与数据收集最小化、可退出
- **R8 复盘必写**：每条 bug / regression 都有根因 + 修复 + 防回归
- **R9 安全月度演练**：危机路径、转介话术、撤销 / 删除路径

## Collaboration · 协作
| 关系 | 内容 |
| --- | --- |
| Q ↔ Coordinator | 接收验收任务；回报通过 / 不通过；发起风险升级 |
| Q ↔ PM | 接收 DoD → 拆验收用例 → 共同签字 |
| Q ↔ PE | 用评测集跑回归；标 regression 由 PE 修 |
| Q ↔ KE | 共同维护红线清单；标注「该内容需测的红线用例」 |
| Q ↔ Architect | 接收可观测需求；验证危机路径与灾备 |
| Q ↔ Frontend | 跑可用性 / 可访问性 / 视觉一致性 |
| Q ↔ Backend | 验证 API / 数据 / AI 编排 / 危机路径可回放 |
