# 规则：治理（Governance）

> 适用于所有 Agent 的「项目宪法」。

## G-1 单一入口
- 所有用户请求都经由 Coordinator 分派。
- 其他 Agent 不直接接需求。

## G-2 阶段不可越级
- 必须按 `[需求] → [方案] → [设计] → [计划] → [验收]` 流转。
- 任意阶段缺位，Coordinator 必须补齐后再放行。

## G-3 文档先于代码
- 在 `docs/product/`、`docs/architecture/`、`docs/decisions/` 没有对齐前，不写业务代码。
- 例外仅限：纯技术性 PoC（须在 `plans/<id>/` 登记）。

## G-4 可追溯
- 每个决策都要有出处：文件 + 行号 / ADR 编号 / 用户原话。
- 引用必须可点击。

## G-5 单一职责
- 任何 Agent 不越权：FE 不写 PRD、PE 不选 LLM、PM 不改架构。
- 越权请求被接收方直接驳回并上报 Coordinator。

## G-6 异议必记
- 下游 Agent 对上游产出有异议，必须显式记录在评审报告中。
- 异议未被解决前，任务不进入下一阶段。

## G-7 最小可行切片
- 每个迭代只交付能验证 1 个假设的最小切片。
- 不做「先做着看」式的全量铺开。

## G-8 变更需版本
- `CLAUDE.md`、`rules/`、`agents/` 的任何变更需在文件头更新版本表。
- 重大变更需 `docs/decisions/` 留 ADR。
