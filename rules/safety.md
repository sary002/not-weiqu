# 规则：安全（Safety）

> 一切功能、对话、内容、数据的「最高优先级」。

> 用户人群是「讨好型人格 / 边界敏感人群」，他们更容易被：
> - 评价性语言刺伤
> - 强推进式交互二次伤害
> - 错误归因（如「你就是太软弱」）进一步自责
> - 隐私泄露加剧社交恐惧
>
> 本文件是产品设计阶段的**最高约束**。

## S-1 绝不替代专业
- 任何 AI 输出都必须带：**「不委屈是自助式成长陪伴，不替代专业心理咨询与诊疗。」**
- 出现疑似精神疾病急性期描述 → 一键转介到专业资源。

## S-2 危机响应（红线）
以下任一情况触发，系统必须**立即停下对话主线**并给出转介：
1. 用户表达自伤 / 自杀 / 他伤意图
2. 急性情绪崩溃 / 惊恐发作
3. 现实中的家暴、跟踪、性侵
4. 未成年人倾诉
5. 妊娠 / 产后阶段出现严重情绪问题

转介内容必须包含：心理援助热线、就近医院精神科、本地紧急电话。
（PE + Q 共同维护转介话术库。）

## S-3 隐私最小化
- 默认不收集：姓名、手机号、身份证、地理位置、通讯录、社交账号
- 任何新字段的引入必须 A 写 ADR + PM 评估必要性
- 第三方 SDK / 分析工具引入前必须过 A + Q

## S-4 语气红线
- ❌「你应该…」「你必须…」「你连这都做不到？」
- ❌「算了吧，别想太多」「忍一忍就过去了」
- ❌ 对用户第三方使用侮辱、贬低、威胁式语言
- ❌ 把边界练习描述为「打败对方」
- ✅ 共情先行 → 命名 → 给出可执行的小动作 → 让用户自决

## S-5 内容红线
- ❌ 任何形式的诊断名词
- ❌ 星座 / 玄学 / 伪科学「疗法」
- ❌ 未经授权的「专家」「机构」背书
- ❌ 引用未注明来源的心理学断言

## S-6 设计红线
- ❌ 用爆红、抖动、惊叹号堆叠制造焦虑
- ❌ 强弹窗、强推送、强打断
- ❌ 「连续打卡 7 天解锁」式道德绑架
- ✅ 用户随时可退出、可暂停、可删除
- ✅ 默认匿名 / 默认最小化

## S-7 升级路径
| 信号 | 处置 |
| --- | --- |
| 用户表达「算了不聊了」 | 平静收束，不再追问 |
| 用户 24h 内多次进入危机话术 | Q 触发人工复核 |
| 用户提及具体可识别第三方 | 提示「可使用代号」并脱敏 |
| 用户付费要求个性化深聊 | 不提供；建议线下咨询 |

## S-8 安全评审
- 任何含情绪触发点的功能上线前，必须过 Q 的红线用例。
- 任何与 LLM 提示词相关的变更，PE 必须自测危机话术不漏触发。
