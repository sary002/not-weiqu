# 规则：质量（Quality）

> 「什么叫做完了」的统一口径。

## Q-1 通用
- **Definition of Done**：
  1. 产出物落盘到约定路径
  2. 自检清单全部勾选
  3. 至少一位同阶段 Agent 旁审
  4. Q 已跑通红线用例

## Q-2 PRD 质量
- 每个 User Story 至少包含：用户 / 场景 / 动机 / 动作 / 验收
- 每个功能至少绑定 1 个可证伪假设
- 每个功能至少标注 1 个「不做的事」

## Q-3 提示词质量
- 语气不漂移评测：通过
- Guardrails 触发率 100%
- 兜底话术评测：通过
- Token 预算评估：A 已签字

## Q-4 内容质量
- 每条知识有来源
- 难度、风险、适用人群已标注
- 至少 2 名 Agent 旁审

## Q-5 代码质量（开发阶段启用）
- 单测覆盖核心路径 ≥ 80%
- 集成测试覆盖跨服务调用
- Lint / Format 通过
- 无 P0 / P1 安全漏洞

## Q-6 文档质量
- 标题 / 一句话目标 / 上下文 / 决策 / 影响
- 链接全部可达
- 版本号与变更记录

## Q-7 评审纪律
- 评审报告必须含：共识 / 异议 / 阻塞 / 建议
- 异议不解决不下放下一阶段
