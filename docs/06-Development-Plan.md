## 1. 目标与读者

把「从 M0 治理到 M5 正式上线」的可执行计划铺平，让 8 个 Agent 与未来的真实团队都能照表走。

读者：C、PM、A、BE、FE、PE、KE、Q，以及未来加入的真实工程师。

## 2. 范围与非范围

### 2.1 范围（MVP · v2.0 形态）
- Web 端（React）
- **底部 4 Tab**：今日（技能树 + 今日三件小事） / 自由对话 / 我的剧本 / 我（进度 + 设置 + 知识小卡）
- **新模块**（v2.0 引入）：技能树（边界成长路 5 段 × N 微技能）、12 步固定流程演练、温和连击 + 休整日、低压力模式
- **保留模块**（v1.0 → v2.0 形态保留）：Onboarding（4 步 + earn-the-commitment）、自由对话（兜底式陪聊）、危机兜底、我的剧本
- 边界五段（L1 觉察 / L2 命名 / L3 表达 / L4 兜底 / L5 巩固）的微技能库（M2 上线 L3 表达 1 个 + L1 觉察 1 个）
- 1 套危机话术与转介资源
- 基础统计与脱敏数据导出 / 删除

### 2.2 非范围（MVP 不做）
- App
- 社区
- 真人对接
- 多模态
- 国际化
- **排行榜 / 打卡天数 / 货币经济 / 付费修复 / 催促型推送**（v2.0 明确拒绝）
- **心数 / 命数 / 倒计时 / 限时折扣**（v2.0 明确拒绝）

> 详见 `docs/01-PRD.md` 第 11 章。

## 6. 依赖图

```
PRD ──► 原型 ──► FE 实施
  │              ▲
  ├──► 架构 ────┤
  │     │       │
  │     ├──► 数据库 ──► API ──► BE 实施
  │     │                     ▲
  │     └──► AI 编排          │
  │           ▲               │
  ├──► 知识块 ┘               │
  │                           │
  └──► 人格 ──► 评测集 ──► Q 验收
                              │
                  部署 ◄──────┘
```

> 关键路径：PRD → 架构 → 数据库 → API → BE 实施 → 集成 → 验收 → 部署。
> 次关键路径：知识块、人格可与架构并行，但注入到 API 必须等接口定稿。

## 14. v2.0 同步说明（本版新增）

### 14.1 v2.0 新增 WBS 来源
- WBS-201 ~ WBS-205 = 5 个 v2.0 全新模块

### 14.2 v2.0 同步的拒绝清单自检项（实施时必跑）
- 代码库无 `heart` / `lives` / `gem` / `coin` / `streak-repair` 字段
- 代码库无 `leaderboard` / `league` / `rank` 组件
- UI 字符串无 "失去" / "丢失" / "再不来就..." / "🔥" 红字
- 推送文案不含 "你快没了" / "再不来就..." / "🔥 火焰"
- 视觉稿中无"红色感叹号 / 抖动 / 闪烁"动效
- 任何路径上都没有"付费墙出现在演练步骤中"
