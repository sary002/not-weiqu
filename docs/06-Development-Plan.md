## 1. 目标与读者

把「从 M0 治理到 M5 正式上线」的可执行计划铺平，让 8 个 Agent 与未来的真实团队都能照表走。

读者：C、PM、A、BE、FE、PE、KE、Q，以及未来加入的真实工程师。

## 2. 范围与非范围

### 2.1 范围（MVP）
- Web 端（React）
- 7 大模块：Onboarding、今日练习、自由对话、危机兜底、我的剧本、进度复盘、设置
- 8 个场景子集
- 1 套危机话术与转介资源
- 基础统计与脱敏数据导出 / 删除

### 2.2 非范围（MVP 不做）
- App
- 社区
- 真人对接
- 多模态
- 国际化
- 排行榜 / 打卡

> 详见 `docs/01-PRD.md` 第 11 章。

## 6. 依赖图

```
PRD ──► 原型 ──► FE 实施
  │              ▲
  ├──► 架构 ────┤
  │     │       │
  │     ├──► 数据库 ──► API ──► BE 实施
  │     │                     ▲
  │     └──► AI 编排          │
  │           ▲               │
  ├──► 知识块 ┘               │
  │                           │
  └──► 人格 ──► 评测集 ──► Q 验收
                              │
                  部署 ◄──────┘
```

> 关键路径：PRD → 架构 → 数据库 → API → BE 实施 → 集成 → 验收 → 部署。
> 次关键路径：知识块、人格可与架构并行，但注入到 API 必须等接口定稿。

## 7. 质量门

| Gate | 阶段 | 准入条件 | 不通过处置 |
| --- | --- | --- | --- |
| G1 | M0 → M1 | 8 Agent + 4 规则 + 9 设计文档齐 | 补齐再进 |
| G2 | M1 → M2 | L2 评审通过 + 红线用例已建立 | 阻塞 |
| G3 | M2 → M3 | PoC 端到端 + 危机话术演练通过 | 阻塞 |
| G4 | M3 → M4 | 封闭用户 ≥ 100 用研通过 + 监控告警就绪 | 阻塞 |
| G5 | M4 → M5 | 渗透测试通过 + SLA 达标 | 阻塞 |
| G6 | 任意阶段 | 危机话术演练不过 | 立即回滚 / 修复 |

## 8. 风险与缓解

| 风险 | 等级 | 缓解 |
| --- | --- | --- |
| 知识块扩量跟不上场景需求 | P0 | M1 即启动 50 条 PoC，M3 到 200+ |
| LLM 供应商波动 | P0 | 主备 + 兜底话术 |
| 危机话术漏触发 | P0 | 保守模型 + 季度演练 |
| 设计反复导致返工 | P1 | L2 评审硬门 |
| 跨 Agent 沟通成本 | P1 | C 主导 + 24h 响应 + 异议必记 |
| 招募真实用户困难 | P1 | 提前 1 个月与社区/学校/企业合作 |
| 性能不达标 | P1 | M4 提前压测 |
| 第三方 SDK 引入 PII | P1 | 0 默认开启 + 审计 |
