## 1. 目标与读者

让用户在「焦虑 / 委屈 / 自我怀疑」的状态下，仍然能：
- 30 秒内找到入口
- 1 分钟内完成一次轻量练习
- 任何时候可退出 / 暂停 / 删除

读者：FE / BE / PE / KE / Q / C。

## 2. 设计原则

> 来自 `agents/frontend.md` 的 R1~R10，全部硬约束。

| # | 原则 | 反例 | 正例 |
| --- | --- | --- | --- |
| P1 | 安静 > 热闹 | 爆红 banner、抖动按钮 | 留白 + 浅灰 + 圆角 |
| P2 | 留白 > 堆叠 | 满屏功能块 | 单列 + 卡片间距 ≥ 16px |
| P3 | 轻交互 > 重交互 | 5 步表单 | 1 步完成，可跳过 |
| P4 | 可中断 > 强推进 | 强弹窗「继续练习」 | 退出 / 暂停 / 保存 三按钮全可达 |
| P5 | 软错误 > 红叹号 | 「错误！」 | 「我们没听清，再说一次？」 |
| P6 | 真人感 > 工具感 | 「操作失败」 | 「我们在这，随时回来」 |
| P7 | 视觉与人格同源 | 字号、断句与 PE 不一致 | 对话气泡的视觉节奏由 PE 签字 |
| P8 | 动效可关闭 | 默认开启动效 | 尊重「减少动效」系统设置 |
| P9 | 不依赖颜色 | 状态仅靠颜色传达 | 颜色 + 文字 + 图标 |
| P10 | 默认匿名 | 主动展示头像 / 昵称 | 用户不输入也能完成全部流程 |

## 3. 视觉语言

### 3.1 色彩
| 角色 | 颜色 | 用途 | 对比度 |
| --- | --- | --- | --- |
| Primary | `#4F5BFF`（温和靛蓝） | 主按钮、链接、关键提示 | AA |
| Primary-Soft | `#EEF0FF` | 卡片底、tag 底 | AA on text |
| Neutral-900 | `#1F2328` | 正文 | AA on white |
| Neutral-500 | `#6B7280` | 次要文本 | AA on white |
| Background | `#FAFAF7`（暖白） | 全局背景 | — |
| Surface | `#FFFFFF` | 卡片 | — |
| Success | `#1F9D55` | 成功提示（不刺眼） | AA |
| Warning | `#B45309` | 软警告 | AA |
| Danger | `#B91C1C` | 仅用于「确认删除」类不可逆操作 | AA |

> 拒绝使用：纯红、亮黄、霓虹色、警示性 emoji 堆叠。

### 3.2 字体
| 角色 | 字体 | 用途 |
| --- | --- | --- |
| 主字体 | `PingFang SC` / `Hiragino Sans GB` / `system-ui` | UI / 正文 |
| 等宽 | `JetBrains Mono` / `Menlo` | 用户脚本片段、错误码 |
| 字号 | 12 / 14 / 16 / 18 / 20 / 24 / 32 | 严格 7 级 |
| 行高 | 1.5（正文）/ 1.7（长文） | — |
| 字重 | 400 / 500 / 600 | 不使用 700+ |

### 3.3 间距 / 圆角 / 阴影
| 项 | 值 |
| --- | --- |
| 间距 | 4 / 8 / 12 / 16 / 24 / 32 / 48 |
| 圆角 | 8（按钮）/ 12（卡片）/ 16（弹窗） |
| 阴影 | `0 1px 2px rgba(0,0,0,.04)`、`0 4px 12px rgba(0,0,0,.06)` |

## 6. 关键流程

### 6.1 今日练习
```
进入首页 → 选场景（选 / 换 / 自定义） → 进入演练
   ↓
演练中（可退出 / 暂停 / 保存）→ 演练后 1 句可执行话术
   ↓
保存到「我的剧本」 / 丢弃 / 改写
```

### 6.2 自由对话
```
进入对话 → 用户输入 → AI 接住 → 命名 → 1 个可执行小动作
   ↓
≤ 6 轮：自由                   ≥ 6 轮：AI 主动收束
   ↓
异常：用户长时间不输入 → AI 主动收束
```

### 6.3 危机响应
```
AI 检测 → 触发危机信号 → 立即停止主线
   ↓
呈现兜底页（热线 / 医院 / 紧急联系人建议）
   ↓
用户选择「我现在安全」 → 回到收束态
   ↓
用户选择「我想继续聊」 → 转人工 / 退出 / 留联系方式
   ↓
事件进入 Q 审计（按 F-CRI-005）
```

## 7. 状态机

### 7.1 全局对话状态机
```
[IDLE]
   ↓  进入对话 / 演练
[ACTIVE]
   ├─ 危机信号 → [CRISIS]   （不可跳过，必须呈现兜底页）
   ├─ 用户主动退出 → [IDLE]
   ├─ 6 轮收束 → [ENDING]  （不催继续）
   └─ 模型超时 → [FALLBACK]（呈现兜底话术）

[CRISIS]
   ├─ 用户确认安全 → [ENDING]
   └─ 用户想继续 → [CRISIS_CHAT]（Q 审计开启）

[ENDING]
   ↓  用户主动开始新会话
[IDLE]
```

### 7.2 演练页状态机
```
[INTRO] → [ROLEPLAY] → [SCRIPT_PROMPT] → [DONE]
   ↓           ↓              ↓             ↓
 退出        暂停           改写          保存
   ↓           ↓              ↓             ↓
[IDLE]     [SAVED]       [IDLE]      [SAVED]
```

## 8. 组件库

| 组件 | 用途 | 关键约束 |
| --- | --- | --- |
| `<Button />` | 主/次/文字 三态 | 主按钮同一页 ≤ 1 个 |
| `<Card />` | 内容容器 | 阴影 + 圆角 12 |
| `<Input />` | 单行输入 | 字数提示，但不强制 |
| `<Textarea />` | 长输入 | 自适应高度 |
| `<Dialog />` | 模态 | 强制可关闭 + 焦点陷阱 |
| `<Toast />` | 非阻塞提示 | 3s 自动消失，可关闭 |
| `<Tag />` | 分类标签 | 软色，不饱和 |
| `<EmptyState />` | 空态 | 永远有「下一步」入口 |
| `<CrisisBanner />` | 危机兜底 | 全产品一致，不可被主题覆盖 |
| `<ProgressCurve />` | 复盘曲线 | 不展示打卡天数 |

## 9. 文案规范

| 场景 | 禁止 | 推荐 |
| --- | --- | --- |
| 拒绝 | 「拒绝」 | 「我可以不做 / 这次不行」 |
| 等待 | 「请稍候」 | 「我们在想」 |
| 错误 | 「操作失败」 | 「我们没听清，再说一次？」 |
| 鼓励 | 「你真棒」 | 「多了一次练习」 |
| 推送 | 「别忘了练习」 | 「想练就练，不想也行」 |
| 危机 | 「你没事吧」 | 「你现在不太好。我们在这。」 |
| 升级 | 「立刻寻求帮助」 | 「建议：拨打热线 / 联系你信任的人 / 就近医院」 |

> 全产品文案须与 PE 的人格定义保持一致，由 PE 签字。

## 10. 微交互与动效

- 加载：骨架屏，禁止使用旋转 loader
- 切换：fade + 4px 位移，150ms
- 危险操作：二次确认，不使用抖动
- 危机：禁用所有动效（避免增加刺激）

## 11. 可访问性

| 维度 | 要求 |
| --- | --- |
| 文字对比度 | ≥ 4.5:1（大字 ≥ 3:1） |
| 键盘可达 | 关键路径 100% Tab 可达 |
| 屏幕阅读器 | 全部组件带 ARIA |
| 焦点指示 | 清晰可见，不依赖颜色 |
| 动效 | 默认尊重「减少动效」系统设置 |
| 字号 | 用户可放大到 200% 不破版 |
| 输入 | 支持纯键盘输入 + 语音输入（可选） |
| 颜色 | 状态不只靠颜色，配文字 / 图标 |

## 12. 响应式与适配

| 断点 | 宽度 | 适配 |
| --- | --- | --- |
| `xs` | < 480 | 单列，触控优先 |
| `sm` | 480~768 | 单列 |
| `md` | 768~1024 | 单列 + 侧栏收起 |
| `lg` | 1024~1440 | 单列 + 侧栏常驻 |
| `xl` | > 1440 | 居中，最大宽度 720 |

> 对话类页面最大宽度 ≤ 720，避免「眼神涣散」。

## 13. 性能预算

| 维度 | 预算 |
| --- | --- |
| 首屏 LCP | ≤ 1.5s（强网）/ ≤ 3s（弱网） |
| 打包体积 | 路由级 ≤ 200KB（gzip） |
| 长对话渲染 | 1000 消息内不卡顿 |
| 资源 | 默认不加载第三方字体 / 跟踪脚本 |

## 14. 与 PE 的协作节奏

| 项 | 节奏 |
| --- | --- |
| 对话气泡字号 / 行高 | FE 出初值 → PE 签字 |
| 长对话折叠 | FE 出阈值 → PE 审语气 |
| 兜底话术 | PE 主写，FE 落组件 |
| 「可执行小动作」展示 | PE 定长度，FE 定视觉 |
| 危机页文案 | PE + KE + Q 三方签字 |
