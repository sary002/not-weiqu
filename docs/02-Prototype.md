## 1. 目标与读者

让用户在「焦虑 / 委屈 / 自我怀疑」的状态下，仍然能：
- 30 秒内找到入口
- 1 分钟内完成一次轻量练习
- 任何时候可退出 / 暂停 / 删除

并且：
- 通过**有限步骤的演练**把"想说但说不出口"翻译成"我可以这样说"——而不只是"倾听了 AI 的回应"。
- 通过**技能树 + 进步可视化**让"成长"变成可被看见的过程，而不是模糊的感受。
- 通过**每日轻量循环**把练习做成习惯，而不是一次性活动。

读者：FE / BE / PE / KE / Q / C。

## 2. 设计原则

> 来自 `agents/frontend.md` 的 R1~R10，全部硬约束。本版在 R2 之上叠加一条 R0：

| # | 原则 | 反例 | 正例 |
| --- | --- | --- | --- |
| **R0** | 成长可被看见 | 模糊感受「我好像有变化」 | 技能树点亮、累计次数、今日三件事完成度 |
| P1 | 安静 > 热闹 | 爆红 banner、抖动按钮 | 留白 + 浅灰 + 圆角 |
| P2 | 留白 > 堆叠 | 满屏功能块 | 单列 + 卡片间距 ≥ 16px |
| P3 | 轻交互 > 重交互 | 5 步表单 | 1 步完成，可跳过 |
| P4 | 可中断 > 强推进 | 强弹窗「继续练习」 | 退出 / 暂停 / 保存 三按钮全可达 |
| P5 | 软错误 > 红叹号 | 「错误！」 | 「我们没听清，再说一次？」 |
| P6 | 真人感 > 工具感 | 「操作失败」 | 「我们在这，随时回来」 |
| P7 | 视觉与人格同源 | 字号、断句与 PE 不一致 | 对话气泡的视觉节奏由 PE 签字 |
| P8 | 动效可关闭 | 默认开启动效 | 尊重「减少动效」系统设置 |
| P9 | 不依赖颜色 | 状态仅靠颜色传达 | 颜色 + 文字 + 图标 |
| P10 | 默认匿名 | 主动展示头像 / 昵称 | 用户不输入也能完成全部流程 |

> **R0 是 Duolingo 借鉴带来的新约束**：练习必须有结构、有节点、有累积，让"变稳了"是一个**可被指认**的事实，而不是自我安慰。

## 3. 形态定位与 Duolingo 借鉴策略

### 3.1 为什么参考 Duolingo

Duolingo 解决的是"**一件本该高频练习但人很难坚持的事**"。不委屈同样如此——边界不是被讲明白的，是被一次次"敢说出口"练出来的。两者面对的认知-行为鸿沟高度同构：

| 维度 | Duolingo | 不委屈 |
| --- | --- | --- |
| 知识缺口 | 知道单词 ≠ 能在真实场景说出 | 知道「要拒绝」≠ 真能平静拒绝 |
| 行为阻力 | 怕错、怕忘、怕坚持不下去 | 怕冲突、怕关系破裂、怕愧疚 |
| 期待结果 | 流利对话 | 在真实场景里"说得出、稳得住" |
| 复访动力 | 每日轻量、可见进步 | 每日轻量、可见进步 |
| 用户脆弱面 | 怕失败、怕丢 streak | 怕被拒绝、怕破坏关系、怕复发 |

### 3.2 三分类决策表

> 把 Duolingo 的设计要素逐条过一遍。**借鉴**=可直接迁移；**改造**=要按本产品伦理与气质重写；**拒绝**=触红线，绝对不引入。

| Duolingo 元素 | 决策 | 在不委屈的形态 |
| --- | --- | --- |
| 技能树 / Learning Path | ✅ **借鉴** | 「边界成长路」5 大区 × 若干微技能节点 |
| 有限步关卡（10–15 步） | ✅ **借鉴** | 「一次演练」= 12 步固定流程，单步单屏单任务 |
| 状态节点图标（锁/可学/进行中/掌握） | ✅ **借鉴** | 技能节点的视觉态：灰锁 / 金边 / 蓝环 / 金冠 |
| 每日目标（10/15/20/30 XP） | ✅ **改造** | 每日「轻 / 稳 / 深入」三档选择（1 / 2 / 3 件事） |
| Earn-the-commitment（先体验后注册） | ✅ **借鉴** | 首次演练完成后才提示是否保存进度 |
| 进步可视化（time-spent / XP-by-skill） | ✅ **改造** | 累计「练过的瞬间」、各区掌握度、本周主题曲线 |
| 连击 Streak | ⚠️ **改造** | 「温和连击」：缺失静默归零 + 每周休整日 + 无付费修复 |
| 每日任务 Quests | ⚠️ **改造** | 「今日三件小事」：1 必选 + 2 选做，无宝石奖励 |
| 推送召回 + 频率档位 | ⚠️ **改造** | 推送默认全关；开启后才出现；带"低压力模式"开关 |
| 心数 Hearts / 命数 Lives | ❌ **拒绝** | 演练无错就扣机制；鼓励试错；暂停可随时按 |
| 宝石 Gems / 货币经济 | ❌ **拒绝** | 全产品无货币、无抽卡、无"攒币换 X" |
| Streak Repair 付费修复 | ❌ **拒绝** | 不存在付费修复；不写"丢失"叙事 |
| 排行榜 League | ❌ **拒绝** | 无任何用户间排名 / 对比 |
| 打卡天数 "X 天挑战" | ❌ **拒绝** | "我练了 X 次"，不写"坚持 X 天"作为价值判断 |
| 推送中的内疚 / 紧迫话术 | ❌ **拒绝** | 邀请式："想练就练，不想也行" |
| 排行榜红点 / 排名下降提醒 | ❌ **拒绝** | 无 |
| 倒计时 / 限时折扣 | ❌ **拒绝** | 无 |
| Path 红点 / 任务红点 | ❌ **拒绝** | 不以"未完成"作为视觉焦点（避免未读焦虑） |

### 3.3 形态对比

| 维度 | v1.0（聊天陪伴） | v2.0（刻意练习） |
| --- | --- | --- |
| 主页 | 列表式入口 | 技能树（垂直路径 + 节点） |
| 核心动作 | 进入对话 | 完成一次有限步演练 |
| 反馈环 | 倾诉 → AI 接住 | 演练 → 收获 1 句可执行话术 → 可存 |
| 进步感 | 月度曲线 | 节点点亮 + 累计瞬间 + 区段掌握度 |
| 复访触发 | 自由倾诉欲望 | 今日三件小事 / 节点解锁 |
| 边界 | 模糊（聊到哪算哪） | 清晰（一次演练 = 12 步，到点收束） |
| 伦理风险 | 强对话可能滑向危机 / 依赖 | 演练收束、危机转介、温和连击 |

> v2.0 保留了 v1.0 的「自由对话」「危机兜底」「我的剧本」三件大事，只把入口形态和主路径重塑。

## 4. 视觉语言

### 4.1 色彩

| 角色 | 颜色 | 用途 | 对比度 |
| --- | --- | --- | --- |
| Primary | `#4F5BFF`（温和靛蓝） | 主按钮、链接、关键提示 | AA |
| Primary-Soft | `#EEF0FF` | 卡片底、tag 底 | AA on text |
| Neutral-900 | `#1F2328` | 正文 | AA on white |
| Neutral-500 | `#6B7280` | 次要文本 | AA on white |
| Background | `#FAFAF7`（暖白） | 全局背景 | — |
| Surface | `#FFFFFF` | 卡片 | — |
| Success | `#1F9D55` | 成功提示（不刺眼） | AA |
| Warning | `#B45309` | 软警告 | AA |
| Danger | `#B91C1C` | 仅用于「确认删除」类不可逆操作 | AA |
| **Skill-Mastered** | `#C9A24B`（暖金） | 技能节点「已掌握」态 | AA on white |
| **Skill-Progress** | `#4F5BFF`（= Primary） | 技能节点「进行中」环 | AA on white |

> 拒绝使用：纯红、亮黄、霓虹色、警示性 emoji 堆叠。
> **新增约束**：暖金仅用于"已掌握"，不用于"胜利 / 击败"，避免对用户形成"战斗"隐喻。

### 4.2 字体

| 角色 | 字体 | 用途 |
| --- | --- | --- |
| 主字体 | `PingFang SC` / `Hiragino Sans GB` / `system-ui` | UI / 正文 |
| 等宽 | `JetBrains Mono` / `Menlo` | 用户脚本片段、错误码 |
| 字号 | 12 / 14 / 16 / 18 / 20 / 24 / 32 | 严格 7 级 |
| 行高 | 1.5（正文）/ 1.7（长文） | — |
| 字重 | 400 / 500 / 600 | 不使用 700+ |

### 4.3 间距 / 圆角 / 阴影

| 项 | 值 |
| --- | --- |
| 间距 | 4 / 8 / 12 / 16 / 24 / 32 / 48 |
| 圆角 | 8（按钮）/ 12（卡片）/ 16（弹窗） / **20（技能节点）** |
| 阴影 | `0 1px 2px rgba(0,0,0,.04)`、`0 4px 12px rgba(0,0,0,.06)` |

### 4.4 技能节点视觉态

> 来自 Duolingo「Lesson node appearance encodes state」的迁移。状态由**颜色 + 图标 + 文字**三重传达，符合 R9。

| 状态 | 颜色 | 图标 | 文字 |
| --- | --- | --- | --- |
| 锁定 | 灰 + 半透明 | （无锁，v2.0.1 polish） | hover 时显示「完成 §X 解锁」 |
| 可学习 | 暖白底 + 金边 | ✦ | 「可练」 |
| 进行中 | 暖白底 + 蓝环 + 进度 | ◐ | 「练到第 N/12 步」 |
| 已掌握（基础） | 暖白底 + 1 颗金冠 | ★ | 「练过 1 次」 |
| 已掌握（深入） | 暖白底 + 2 颗金冠 | ★★ | 「练过 N 次」 |
| 当前推荐 | 暖白底 + 金边 + 软发光 + 微微放大 1.05x | ✦ | （无"今日"文字，v2.0.1 polish） |

> 节点之间用"路径"（曲折细线）连接，呼应 v1.0 P6「旅程感」。
> **v2.0.1 polish 备注**：
> - 锁定节点不显示 🔒——锁对脆弱用户是"被拒绝"的感觉；改用半透明 + hover 引导
> - "今日推荐"删除文字标签——Jobs 风格通过视觉（发光 + 放大）传达，不解释

## 5. 信息架构

### 5.1 顶层导航（Web + App）

**底部 Tab · 4 项**（参考 Duolingo「action-first」原则，删除复盘为独立 Tab）：

```
┌──────────────────────────────────────────────┐
│  🏠 今日  ｜  💬 自由对话  ｜  📖 我的剧本  ｜  👤 我  │
└──────────────────────────────────────────────┘
```

| Tab | 主要任务 | 与 v1.0 的差异 |
| --- | --- | --- |
| 🏠 今日 | 技能树 + 今日三件小事 | 替代"今日练习 + 自由对话"双入口 |
| 💬 自由对话 | 倾诉与命名（兜底式） | 升级为独立 Tab，更克制 |
| 📖 我的剧本 | 沉淀话术 | 名称不变，地位提升 |
| 👤 我 | 进度 + 设置 + 知识小卡 | 收纳复盘、设置、小卡为一处 |

> 复盘不再是独立 Tab，而是「我 → 进度」子页（避免 v1.0 那种"被要求看自己"的压力）。

### 5.2 全局可达
- 顶部右上：退出 / 暂停对话（任何页面可达）
- 底部常驻：「随时退出」「不替代专业」
- 顶部左侧：低压力模式开关（默认开）
  - 开启时：静默所有推送 / 静默所有红点 / 节点无闪烁
  - 关闭时：可订阅"今日完成"轻提示（用户主动开启，绝不默认开）

### 5.3 技能树结构

> 来自 Duolingo「Section → Unit → Skill → Lesson」层级的迁移。本产品对齐 `docs/01-PRD.md` §7「边界五层模型」。

```
边界成长路（Learning Path）
├─ §1 觉察 · L1          ← 看见自己
│  ├─ 听见身体的信号
│  ├─ 给情绪命名
│  └─ 看见讨好模式
├─ §2 命名 · L2          ← 说出自己
│  ├─ 说出"我想要"
│  ├─ 说出"我不想"
│  └─ 区分"我的 / 别人的"
├─ §3 表达 · L3          ← 在真实场景里说  ← 核心区，多数用户主战场
│  ├─ 拒绝同事甩活       ← 经典开场
│  ├─ 拒绝加班到深夜
│  ├─ 拒绝朋友借钱
│  ├─ 春节应对亲戚追问
│  ├─ 应对伴侣冷暴力
│  └─ 在服务场景不卑不亢
├─ §4 兜底 · L4          ← 被回绝后不崩塌
│  ├─ 被回绝后自我安抚
│  ├─ 应对愧疚感
│  └─ 在冷暴力里站稳
└─ §5 巩固 · L5          ← 把单次胜利变成习惯
   ├─ 整理本周胜利
   ├─ 复习我的剧本
   └─ 30 日回顾
```

每个微技能（Skill）内含 5–7 个**演练**（Lesson），每个演练 = 1 个 12 步固定流程。

### 5.4 今日三件小事

> 来自 Duolingo「Daily Quests」改造。每日 3 件可完成的小事，**无宝石奖励**，只有完成感。

| 类型 | 数量 | 来源 | 用户可否跳过 |
| --- | --- | --- | --- |
| **必做** | 1 | 技能树当前位置（系统推荐） | 必做，但用户可换成"今日休整" |
| **选做 A** | 1 | 复习（系统从「我未掌握的」中挑 1） | 可跳过 |
| **选做 B** | 1 | 知识小卡 OR 自定义场景 | 可跳过 |

- 完成 1 件 = 50% / 2 件 = 100% / 3 件 = 150%（不内卷，超过不奖）
- 用户可在「我 → 设置」中把"今日必做"也设为"选做"
- 三件小事**永远不显示"未完成"红色状态**，只显示「今日 0/3」灰色进度条

### 6.1 Onboarding（不审问式 · Earn-the-commitment）· MVP

> 借鉴 Duolingo 「完成一节课前不要求注册」+ 「先做价值承诺、再做身份承诺」。

```
┌────────────────────────────────────────────────┐
│  Step 1 / 4                  [ 跳过全部 → ]    │
├────────────────────────────────────────────────┤
│                                                │
│  帮善良的人学会有锋芒                          │
│                                                │
│  选一个最贴近你最近的场景：                    │
│   ○ 同事又甩活                                │
│   ○ 家人又催我                                │
│   ○ 朋友又提过分要求                          │
│   ○ 伴侣冷战 / 不回消息                       │
│   ○ 都差不多                                  │
│                                                │
│            [  下一步  ]                        │
│                                                │
│  不委屈是成长陪伴，不替代专业心理咨询。        │
└────────────────────────────────────────────────┘

↓ Step 2（每天想做几件小事？轻/稳/深入，3 选 1）
↓ Step 3（是否开启低压力模式？默认开）
↓ Step 4（注册？先体验，注册可后置）
```

- 步骤 ≤ 4
- 每步 1 个问题 + 1 个主按钮
- 「跳过全部」常驻，**不强制任何问题**
- 推送 / 通知收集在「体验后」询问，而非注册前

## 7. 关键流程

### 7.1 一次完整的"今日"

```
进入今日 → 看技能树 → 看今日三件小事
   ↓
[ 必做 ] 选一个微技能 → 看场景卡 → 12 步演练 → 收束
   ↓
[ 选做 A ] 系统推荐 1 个复习 → 同样的 12 步流程（场景为"我之前没掌握"）
   ↓
[ 选做 B ] 知识小卡 OR 自定义场景
   ↓
回到技能树，今日 0/3 → 1/3 → 2/3 → 3/3
   ↓
任意时刻可点 "今天就到这里" → 直接退出，无惩罚
```

> "今天就到这里" 按钮 = v2.0 的关键设计。**永远不出现"你今天还没练完"提示**。

### 7.2 一次演练

```
进入 INTRO（场景卡）
   ↓
STEP 1-5：自我觉察与命名（5 步必走）
   ↓
STEP 6-8：AI 演练（角色扮演）   ← 可暂停 / 可退出
   ↓
STEP 9-10：沉淀话术
   ↓
STEP 11：保存到我的剧本（可改 / 可跳）
   ↓
STEP 12：收束
   ↓
回到今日，今日进度 +1
```

### 7.3 中断与继续

> 借鉴 Duolingo「Pause anytime」+ 强化。

```
演练中点 [ 暂停 ]
   ↓
弹 3 选项：[ 5 分钟后继续 ] [ 改天继续 ] [ 直接退出 ]
   ↓
状态保存为「草稿」，下次进「今日」会顶部提示"你有一次未完成的演练"
   ↓
点提示 → 回到第 X 步继续（X = 暂停步）
   ↓
"改天继续"超过 3 天 → 草稿归档为"过往的尝试"，**不计入累计**（避免"我那次没练完"成为内疚源）
```

### 7.4 自由对话（保留 v1.0）

```
进入自由对话 → 用户输入 → AI 接住 → 命名 → 1 个可执行小动作
   ↓
≤ 6 轮：自由                   ≥ 6 轮：AI 主动收束
   ↓
异常：用户长时间不输入 → AI 主动收束
```

### 7.5 危机响应（保留 v1.0，路径不变）

```
AI 检测 → 触发危机信号 → 立即停止主线
   ↓
呈现兜底页（热线 / 医院 / 紧急联系人建议）
   ↓
用户选择「我现在安全」 → 回到收束态
   ↓
用户选择「我想继续聊」 → 转人工 / 退出 / 留联系方式
   ↓
事件进入 Q 审计（按 F-CRI-005）
```

### 7.6 温和连击（改造 Duolingo Streak）

> **核心伦理原则**：连击的"快乐"必须大于"丢失的恐惧"。

```
[ 触发 ] 用户完成 ≥ 1 次演练 = 当日算"练过"
[ 计数 ] 连续 N 天都有"练过" = 显示 N（无火焰表情升级，不做颜色升级到 100 天后刺眼）
[ 缺失 ] 缺失 1 天 = 静默归零（不发推送、不弹窗、不写"已丢失"）
[ 休整 ] 每周可标记 1–2 天为"休整日" = 不计入断签
[ 修复 ] 不存在付费修复
[ 文案 ] 顶部小字："练过 N 次 · 连续 N 天 · 每周可休整 1 天"

⚠ 关键：缺失的那天，UI 上**没有任何"丢失"叙事**。
   不出现："你失去了 23 天的连击"
   只出现："下次想练就练"
```

## 8. 状态机

### 8.1 全局产品状态机

> 主体与 v1.0 一致。新增 PRACTICE 子状态机。

```
[IDLE]                            ← 今日主页
   ↓  进入演练
[PRACTICE_INTRO]
   ↓  开始
[PRACTICE_ACTIVE]                 ← 12 步演练中
   ├─ 危机信号 → [CRISIS]   （不可跳过，必须呈现兜底页）
   ├─ 用户暂停 → [PRACTICE_PAUSED]
   ├─ 用户退出 → [IDLE]（草稿保存）
   └─ 12 步完成 → [PRACTICE_DONE]

[PRACTICE_DONE]
   ├─ 用户保存 → [IDLE] + 进度 +1
   ├─ 用户改写 → [PRACTICE_ACTIVE]（回到 STEP 9）
   └─ 用户跳过 → [IDLE]（不计入进度）

[CRISIS]
   ├─ 用户确认安全 → [ENDING]
   └─ 用户想继续 → [CRISIS_CHAT]（Q 审计开启）

[PRACTICE_PAUSED]
   ├─ 继续 → [PRACTICE_ACTIVE]（回到暂停步）
   ├─ 改天继续 → [IDLE]（草稿）
   └─ 直接退出 → [IDLE]（草稿）

[ENDING]
   ↓  用户主动开始新会话
[IDLE]
```

### 8.2 演练内部状态机

```
[INTRO] → [STEP_01] → [STEP_02] → ... → [STEP_12] → [OUTRO]
   ↓           ↓             ↓              ↓            ↓
 退出        暂停           改写           保存       收束
   ↓           ↓             ↓              ↓            ↓
[IDLE]     [PAUSED]     [STEP_09]      [IDLE]      [IDLE]
                                            + 进度 +1
```

### 8.3 温和连击状态机

```
[ACTIVE: 0]        ← 今日未练
   ↓  今日完成 ≥ 1 次
[ACTIVE: 1]        ← 连续 1 天
   ↓  ...每日 +1
[ACTIVE: N]
   ↓  缺失 1 天（非休整日）
[RESET: 0]         ← 静默归零
   ↓  不发推送、不弹窗、不显示"丢失"
   ↓  下次完成 → [ACTIVE: 1]
```

## 11. 微交互与动效

- 加载：骨架屏，禁止使用旋转 loader
- 切换：fade + 4px 位移，150ms
- 危险操作：二次确认，不使用抖动
- 危机：禁用所有动效（避免增加刺激）
- **新增：节点点亮**
  - 状态从「可学习」→「进行中」：圆环由灰渐变到蓝，500ms，无抖动
  - 状态从「进行中」→「已掌握基础」：圆环收束成 1 颗金冠，700ms，无 flash
  - **禁止**：连续闪烁、shake、彩带、爆红
- **新增：温和连击数字变化**
  - +1：数字 fade-in 200ms
  - 归零：无任何动效（避免"丢失"感知）
  - **禁止**：火焰熄灭动画、"你失去了"toast

## 12. 可访问性

| 维度 | 要求 |
| --- | --- |
| 文字对比度 | ≥ 4.5:1（大字 ≥ 3:1） |
| 键盘可达 | 关键路径 100% Tab 可达 |
| 屏幕阅读器 | 全部组件带 ARIA；技能节点状态由文字 + 图标 + ARIA-label 共同传达 |
| 焦点指示 | 清晰可见，不依赖颜色 |
| 动效 | 默认尊重「减少动效」系统设置 + 「低压力模式」开关 |
| 字号 | 用户可放大到 200% 不破版 |
| 输入 | 支持纯键盘输入 + 语音输入（可选） |
| 颜色 | 状态不只靠颜色，配文字 / 图标 |
| **认知负荷** | 一次演练 12 步固定，不做动态长度 |
| **感官敏感** | 危机页与「低压力模式 ON」时**禁用所有非必要动效与声音** |
