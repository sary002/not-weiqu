## 0. 方法学与来源说明(必读)

在阅读本报告前,请先了解以下限制:

1. **外部工具受限**:本研究执行时,`WebSearch` 与 `WebFetch` 工具暂时不可用(API 400 / 域名验证失败)。因此,报告中的事实性陈述**主要来自 LLM 训练知识截止前的可公开检索资料**,并在文末「来源与核验建议」一节给出推荐的人工核验 URL 列表。
2. **核验责任**:报告中带"≈"前缀或"据公开资料"措辞的论断,需由 **PE / KE / PM** 在落地前通过官方资料、学术论文、近期评测至少核验一次。
3. **争议性论断**:Streak 焦虑、Hearts 焦虑、Leaderboard 羞辱、dark pattern 等批评性论断,基于 Wired、UX Collective、ACML 等英文媒体的公开讨论;不同源之间存在分歧,本报告在「不适用/需调整的机制」一节明确标注。
4. **核心立场**:不委屈是一款面向**情绪脆弱用户**的成长产品,所有"借什么、避什么"的取舍,都必须经过「是否会伤害脆弱用户」这一关(对应 `CLAUDE.md` §1 治理原则 §1 与 §5 红线 §6)。

## 1. 执行摘要(Executive Summary)

不委屈可以从三个 app 各取一段**主轴**,但都需要重新设计后才能安全使用:

- **从 Duolingo 借"进度可视化"与"日目标"** 的轻量游戏化,而**不**借其 Streak 焦虑触发、Hearts 失败惩罚、Leagues 排行榜羞辱。Duolingo 的高强度游戏化对语言学习者有效,但放到情绪训练场景,会从"驱动"变成"二次伤害"。
- **从 Ahead 借"教练人格 + 短时微课 + 思维记录(Thought Record)"**,这是与不委屈定位最接近的样本。Ahead 的产品形态(每日 5-10 分钟、教练音频/图文、心情记录、订阅制)是可平移的**内容与节奏模板**;但 Ahead 偏正向(成长)取向,缺少"边界训练"这一具体场景。
- **从 Brilliant 借"引导式发现 + 反馈解释体系"** 的交互范式,直接迁移到不委屈的**对话练习与角色扮演**模块。Brilliant 的 Hint 体系、"为什么错"的解释结构、可视化思考过程,是把"对错判断"转成"反思性学习"的关键。

**最值得立刻实验的 3 个机制(M2/M3 假设)**:

| # | 假设 | 来源 | 实验形态 |
|---|------|------|----------|
| H1 | "每日一题 · 微场景卡片" | Ahead × Brilliant | 每日 1 个 3-5 分钟的边界场景对话练习,完成后给 1 句教练复盘 |
| H2 | "温柔度进度环" | Duolingo(改造) | 进度可视化用"温柔度"等正向词,代替 XP 数值;0 失败惩罚 |
| H3 | "不评判错答"反馈体系 | Brilliant | 答错时,展示"另一种可能"而非"答错"红色标记;引导式发现代替对错判断 |

## 2. Duolingo(多邻国)· 游戏化机制拆解

### 2.1 定位与用户

- **定位**:全球最大语言学习 app(MAU 约 7400 万+,数据为约 2024-2025 年公开数字,**需核验**)。
- **用户**:从青少年到 60+ 退休学习者,主要动机是"低门槛、可持续的小成就感"。
- **商业模式**:Freemium,核心订阅为 **Super Duolingo / Duolingo Max**(AI 对话教练 Max 基于 GPT-4)。

### 2.2 核心机制拆解

| 机制 | 规则 | 设计意图 |
|------|------|----------|
| **Streak(连续打卡)** | 每天完成 1 课保留 streak,中断则归零。Streak Freeze 道具可"冻结" 1 天 | 利用损失厌恶(Loss Aversion)与"沉没成本"促日活 |
| **Hearts(生命值)** | 默认 5 颗心,答错扣 1,扣完则需等待恢复或花宝石续命 | 通过"惩罚"增加专注度;2023 起逐步向"Max Hearts"软化 |
| **XP / 等级** | 完成课程获 XP,累计 10/20/30 天升一阶 | 长期进度感,可视化"努力" |
| **Leagues(排行榜)** | 钻石/蓝宝石/红宝石/翡翠/金/银/铜周赛,30 人一组,前 7 名晋级 | 社交比较驱动;争议较大,被批"羞辱慢学者" |
| **Quests(任务)** | 每日/每周任务(如"今天完成 3 课")奖励宝石 | 细分目标,降低启动成本 |
| **Achievements(成就)** | "早起鸟""完美周""百日连击"等徽章 | 收集欲与里程碑 |
| **Gems / Lingots(宝石)** | 课内货币,用于购买 Streak Freeze、心续命、装扮 | 软经济循环 |
| **每日学习目标(Goal)** | Casual / Regular / Serious / Insane | 用户自选强度,保留掌控感 |
| **学习路径结构** | Section → Unit → Lesson,沿 CEFR 分级(A1-C1) | 长程进度感 |

### 2.3 关键交互与文案

- **关卡结束页**:大号 XP 数字 + "你已完成 X 课" + "继续学习"按钮,持续正反馈。
- **失败/答错**:小振动 + 红色 + "正确答案是 X",即时纠错(对语言学习有效,但对情绪话题**反向**)。
- **推送文案**(业界研究最深的点之一):
  - "别让你的 streak 死掉!🦉"
  - "iOS 提醒:这些通知正在变得愈益咄咄逼人"(Wired 2023)
- **品牌人格**:Duo(猫头鹰吉祥物)以**第二人称调侃、施压、愧疚**闻名。
- **视觉**:明黄绿 + Duo IP,扁平卡片,5-10 分钟微单元。

### 2.4 争议与反思

- Wired、The Guardian、ACML 论文等近年多次批评 Duolingo 的"通知施压"对部分用户造成**焦虑、内疚、依赖**。
- 部分研究指出 Streak 对成人学习者具有正向作用,但对**情绪脆弱人群**会触发"完美主义失败循环"。

> **对不委屈的启示**:游戏化机制是一把**双刃剑**。可借"进度可视化",但需替换所有"惩罚/丢失/比较"语义。

## 3. Ahead(情绪训练 app)· 情绪训练内容拆解

### 3.1 定位与用户

- **定位**:"大脑训练,每日只需几分钟"(原口号,需核验),核心是 **CBT(认知行为疗法) + 正念 + 教练人格化内容**的轻量自助 app。
- **用户**:20-40 岁都市白领,主要为焦虑、压力、缺乏自信、关系困扰(边界、亲密关系)等轻度到中度心理困扰;**不替代专业治疗**。
- **商业模式**:Freemium,免费每天 1 课;订阅解锁全部课程、心情记录、深度训练。
- **创立背景**:由耶鲁 / 斯坦福背景团队创立(具体创始人信息需核验)。

### 3.2 核心机制拆解

| 机制 | 形态 | 设计意图 |
|------|------|----------|
| **每日一课(5-10 分钟)** | 教练音频 + 互动练习 + 总结卡片 | 低门槛,形成日活 |
| **教练人格(Coach Persona)** | 多个"教练"角色,每个有特定声音/语气/价值观 | 选择亲近感,缓解"机器感" |
| **课程分类** | 焦虑、自信、压力、睡眠、关系/边界、习惯等主题 | 模块化,便于个性化推荐 |
| **Thought Record(思维记录)** | CBT 经典工具:情境 → 自动思维 → 情绪 → 替代思维 | 把抽象"心理训练"落到纸面 |
| **Triggers(触发器)** | 课程前先问"你今天想处理什么?",情境化锚定 | 即时相关性,提高迁移 |
| **Mood Tracking(心情追踪)** | 每日心情打卡,周/月曲线 | 行为数据 + 自我观察 |
| **多模态内容** | 音频(教练主讲)+ 文本(卡片)+ 偶尔视频/插图 | 适配不同场景(通勤用听、午休用读) |
| **进度与报告** | 完成课程数、连续天数、心情变化 | 强化"在变好"的感觉 |
| **商业化钩子** | 免费首日,后续付费墙("解锁全部教练 + 全部课程") | 短试 → 长期承诺 |

### 3.3 关键交互与文案

- **课程开头**:教练口播"嗨,今天我们聊聊……"或"你现在感觉怎么样?"(强关系感)。
- **正念引导**:"慢慢吸气……感受脚底接触地面……" (音频为主,文本为辅)。
- **思维记录卡片**:以"对话气泡 + 选择题 + 自由文本"组合,避免"答题"感,鼓励表达。
- **文案语气**:温暖、第二人称、非评判、避免专业术语("CBT 暴露"等),更接近朋友建议。
- **品牌视觉**:柔和粉/米色/浅蓝渐变,圆润插画,无强 IP,营造"安全空间"。

### 3.4 对不委屈最直接可平移的部分

- **每日微课 + 教练人格**:不委屈可以**直接复用此节奏**,把"AI 教练"作为对话主轴。
- **Thought Record**:不委屈的"对话练习"模块可以**采用同款卡片化结构**(情境 → 想法 → 情绪 → 替代表达)。
- **心情追踪**:作为成长报告的输入数据。
- **课程分类**:**边界(Relationships/Boundaries)模块**已经在 Ahead 课程目录中,说明这是有付费意愿的市场。

### 3.5 Ahead 自身的局限

- 内容深度有限,长期留存是问题(行业通病)。
- 教练人格被部分用户批评"过于温柔,缺少推力"。
- 边界主题课程不多(占比估计 < 10%,**需核验**)。

## 4. Brilliant(交互式学习)· 交互式练习拆解

### 4.1 定位与用户

- **定位**:"用问题思考"(Think with problems),强调**主动发现式**学习的 STEM 平台。
- **用户**:好奇心驱动的成人学习者,以 18-35 岁、男性略多(传统印象,需核验近期数据);高学历倾向。
- **商业模式**:Freemium,免费每日挑战,订阅解锁完整课程库。
- **创立背景**:由 CEO Sue Khim 创立(需核验,曾有 Koum 参与早期天使)。

### 4.2 核心机制拆解

| 机制 | 形态 | 设计意图 |
|------|------|----------|
| **Guided Discovery(引导式发现)** | 不直接给结论,让用户先尝试 → 给反馈 → 引导思考 | 主动学习 > 被动阅读;提升迁移率 |
| **多类型交互** | 拖拽、滑动、选择题、填空、自由输入、模拟器 | 适配不同认知任务 |
| **概念可视化** | 大量 SVG / Canvas 动画,让抽象概念"动起来" | 视觉化降低认知负担 |
| **Hint 体系** | 3 档提示:第一档隐晦,第三档接近答案 | 保留探索感,避免"作弊" |
| **错答反馈** | "为什么错"的解释结构:正解 + 错因 + 类比 | 反思 > 评判 |
| **Daily Challenge(每日挑战)** | 每天 1 道新题,跨学科,3-5 分钟 | 长期留存的"日活钩子" |
| **课程结构** | Foundations → Topic Tracks → Courses → Lessons → Problems | 长程路线感 |
| **思考过程可视化** | 部分题目展示"解题路径"或"分支讨论" | 让"思考"本身成为可见对象 |
| **进度与成就** | Streak(温和版,无失败惩罚)、徽章、课程地图 | 传统游戏化,但弱化竞争 |

### 4.3 关键交互与文案

- **题目开头**:大号问题 + 一句话情境 + "想一想" 提示,**不直接展示选项**。
- **答错**:不弹红框,而是显示"接近了,试试这个角度" + 一行解释;再错就给提示。
- **正解**:正反馈 + 短解释 + 立即出下一题(无停顿)。
- **课程结束页**:知识图谱点亮一格,有"你刚学到的" 3 句话总结。
- **文案语气**:中性 + 智识感,鼓励思考而非施压;**没有情绪化施压**。
- **视觉**:深色背景 + 高对比色块,极简,工具感强。

### 4.4 对不委屈最直接可平移的部分

- **Guided Discovery → 引导式反思**:不委屈的对话练习不必"判分",而是**引导用户自己发现更合适的回应**。这与"边界"场景高度契合——边界感不是"教"出来的,是"悟"出来的。
- **Hint 体系 → 教练反馈梯度**:把教练反馈分成 3-5 档,从"只是重述你说的"到"试试这个角度"到"有研究显示……",形成**柔性渐进**的引导,避免一开始就"教育"用户。
- **错答反馈 → "另一种可能"**:用"如果这样回应,可能会发生什么?"代替"答错了"。这是与 Duolingo 最大不同的设计选择。
- **思考过程可视化 → 决策路径展示**:在角色扮演后,展示"其他用户的不同选择"或"这个场景在心理学研究中的常见反应"。

### 4.5 Brilliant 自身的局限

- 强 STEM 取向,**迁移到软性话题(情绪/关系)需要重新设计**。
- 视觉化成本高(动画、SVG),对内容生产是负担。
- 引导式发现需要**内容设计能力极强**,对 PE/KE 角色是考验。

## 5. 可借鉴机制矩阵(15+ 条)

> **难度**:🟢 低 / 🟡 中 / 🔴 高
> **风险等级**:🟢 低(对脆弱用户伤害小)/ 🟡 中 / 🔴 高(可能二次伤害)

| # | 机制名 | 来源 | 适配建议 | 风险提示 | 难度 | 风险 |
|---|--------|------|----------|----------|------|------|
| M1 | **进度可视化(进度环/地图)** | Duolingo | 用"温柔度""自我接纳度"等正向词代替 XP 数值;点亮一格 = 一次练习,不做对错 | 仍可能引发"必须填满"的强迫;需提供"休息日" | 🟢 | 🟡 |
| M2 | **每日自选目标(Casual/Regular/Serious)** | Duolingo | "今天想用 3 分钟 / 5 分钟 / 10 分钟",**默认 Casual** | 选"Serious"的用户可能自我施压 | 🟢 | 🟡 |
| M3 | **课程路径结构(模块→单元→单课)** | Duolingo | 复制 Section → Lesson,加 CEFR 风格"边界级别"(L1 觉察→L2 表达→L3 维护) | 级别标签可能被解读为"我等级低" | 🟡 | 🟡 |
| M4 | **每日微课(5-10 分钟)** | Ahead | 1 节 AI 教练课 + 1 个 Thought Record 卡片 + 1 句复盘,严格 5-10 分钟 | 微课质量参差会让用户失望 | 🟢 | 🟢 |
| M5 | **教练人格(Coach Persona)** | Ahead | 2-3 个教练(温暖姐姐/理性朋友/活泼伙伴),用户选择;**不暴露 LLM 身份** | 人设不一致会让用户失望;需"防爆照"话术 | 🔴 | 🟡 |
| M6 | **Thought Record 卡片** | Ahead | 情境 → 自动想法 → 情绪强度 → 替代表达,卡片式填写 | 触发回忆痛苦场景;需"安全退出"按钮 | 🟡 | 🟡 |
| M7 | **心情追踪(每日打卡)** | Ahead | 1-5 标度 + 1 句"今天发生了什么",曲线展示 | 抑郁期用户可能"打不出来"压力 | 🟢 | 🟡 |
| M8 | **Triggers 情境化(开场问"今天处理什么")** | Ahead | 每次进入先选 1 个场景标签(朋友借钱/家人干涉/同事甩锅),然后定向练习 | 选择困难症;提供"随机给我一个" | 🟡 | 🟢 |
| M9 | **多模态内容(音频/图文/视频)** | Ahead | 教练音频 + 文字总结卡片 + 偶尔视频;通勤可用听 | 视频制作成本高 | 🟡 | 🟢 |
| M10 | **Guided Discovery(引导式发现)** | Brilliant | 不判"对/错",而是"这个回应会让对方感觉……吗?试试……" | 引导要"刚刚好",过度引导会变说教 | 🔴 | 🟢 |
| M11 | **3 档 Hint 体系** | Brilliant | 软提示→思路提示→类比提示;不直接给答案 | 提示过多会变成依赖;限制每日提示数 | 🟡 | 🟡 |
| M12 | **错答反馈的"为什么"解释** | Brilliant | 不说"错",说"如果这样回应,对方可能听到的是……" | 解释过长会让用户读不下去 | 🟡 | 🟢 |
| M13 | **思考过程可视化** | Brilliant | 决策路径展示:"其他人通常这样想/这样说",让用户看到"不只我这样" | 隐私感;不可展示真实用户数据 | 🟡 | 🟢 |
| M14 | **每日挑战(Daily Challenge)** | Brilliant × Duolingo | 每天 1 个新场景对话练习,3 分钟,无失败惩罚 | 强制日活会让疲惫期用户压力大 | 🟡 | 🟡 |
| M15 | **温和版 Streak(连续天数,无丢失惩罚)** | Duolingo(改造) | "你已连续 7 天来看自己"——只记录不惩罚,中断不归零 | 仍可能引发愧疚;考虑改成"累计天数" | 🟢 | 🟡 |
| M16 | **模块化课程库** | Ahead | 主题 × 难度二维矩阵;用户可自由组合"今日课表" | 自由度过高会导致选择瘫痪;需默认推荐 | 🟡 | 🟢 |
| M17 | **安全退出 / 暂停按钮** | Ahead(改造)+ 通用 | 任何时候一键"今天就到这里",无失败反馈 | 容易被滥用逃避;但"逃避"本身可能是健康选择 | 🟢 | 🟢 |
| M18 | **隐私优先的进度报告** | Ahead(改造) | 心情曲线、练习次数**只展示趋势**,不展示具体场景标签;永远不分享 | 错误展示可能"暴露"用户的家庭关系 | 🟡 | 🟡 |
| M19 | **教练反馈的"中性话术"** | Brilliant × Ahead | 用"可以试试"代替"你应该",用"我注意到"代替"你总是" | 团队文化;需写入 `prompts/coach.md` 守则 | 🟡 | 🟢 |
| M20 | **课程完成后的"1 句复盘"** | Ahead(改造) | 不评分,只生成 1 句"今天你看到自己……",镜像用户原话 | 复盘过深会反噬;限制在 30 字内 | 🟢 | 🟢 |

## 6. 不适用 / 需调整的机制

### 6.1 必须**避免**(直接不学)

- ❌ **Hearts / 生命值扣减**(Duolingo):答错扣心 → 在情绪场景中是**直接惩罚**,违反「不评判」原则。
- ❌ **Streak 丢失焦虑**(Duolingo 原始版):用愧疚驱动日活 → 对脆弱用户是二次伤害。
- ❌ **Leagues 排行榜**(Duolingo):社交比较 → 讨好型人格用户本身就更怕"输",排行榜会放大此痛。
- ❌ **"答错"红色标记**(Duolingo):语言学习有效,情绪训练中是评判。
- ❌ **咄咄逼人的推送文案**(Duolingo "Don't break your streak"):**不委屈应**只推送**不催促**,例如"我们在这里,你想来就来"。

### 6.2 必须**弱化**(借鉴框架,降低强度)

- 🔸 **每日目标强度**:默认 Casual,鼓励"做 1 分钟也是做了"。
- 🔸 **成就/徽章**:可保留,但不可与价值挂钩(不能说"达成 X 才是好用户")。
- 🔸 **课程级别标签(A1-C1)**:改为"觉察 → 表达 → 维护 → 修复"4 阶段,**不要数字等级**,避免"我在 L1,真差"。
- 🔸 **游戏化货币(Gems/Lingots)**:不引入虚拟货币,改为"温柔的提示"或"教练的鼓励卡片"。

### 6.3 必须**重新设计**(保留意图,改变形式)

- 🔄 **连续天数**:从"streak" → "累计来过 N 天",**永远不归零**。
- 🔄 **错答反馈**:从"错" → "另一种可能" + 引导式提问。
- 🔄 **排行榜**:从"比赛" → "同伴故事"(匿名分享"其他用户如何处理类似场景")。
- 🔄 **教练人格**:从"完美专家" → "也犯过错的朋友",**适度示弱**,减少权威感。
- 🔄 **推送通知**:从"提醒你做" → "邀请你来";频率上限 1 次/周,**默认关闭**,用户主动开启。

## 7. 合成方案建议 · M2/M3 实验假设

> **治理原则**:每条假设都必须可在 1-2 周内用最简方式验证,且不伤害脆弱用户。

### 假设 H1:每日一题 · 微场景卡片(Ahead × Brilliant)

- **形态**:每天给 1 个 3-5 分钟的边界场景对话练习,完成后给 1 句教练复盘。
- **关键设计**:不评分、不判错、引导式;**今天没做,明天依旧有新题**。
- **验证指标**:
  - 北极星:连续 7 天完成的用户占比(温和版留存)
  - 反指标:课程后 24h 内负面反馈率(需 < 5%)
  - 定性:用户访谈"今天的练习让你感觉……"
- **落地路径**:`agents/pe-prompt-engineer.md` 写 1 个场景脚本 → `docs/prototype/` 加 1 个"微场景卡片"组件 → 5-10 个种子用户跑 1 周。

### 假设 H2:温柔度进度环(Duolingo 改造)

- **形态**:1 个圆形进度环,中心词是"温柔度"或"自我接纳",外圈是"觉察 / 表达 / 维护 / 修复" 4 个刻度;每完成 1 个练习,环上点亮 1 小段。
- **关键设计**:**永远不归零**,只进不退;进度词**不与他人比较**;提供"暂停进度环"选项。
- **验证指标**:
  - 北极星:用户第 14 天主动打开 app 的比例
  - 反指标:用户主动关闭进度环的比例(若 > 30%,说明反而造成压力)
- **落地路径**:Figma 1 个 mock → 前端 1 个组件 → 上线看 100 用户行为。

### 假设 H3:不评判错答的反馈体系(Brilliant)

- **形态**:对话练习中,当用户回应"不太合适"时,展示**两个其他可能** + 1 句"这意味着对方可能感觉到……",而不是"❌ 答错"。
- **关键设计**:每个"错答"也保留(可回顾"我之前说过的话");引导式提问(没有标准答案)。
- **验证指标**:
  - 北极星:用户回看自己过去对话的次数(高 = 反思文化建立)
  - 反指标:用户主动删除对话的次数(若高,说明他们不愿面对)
- **落地路径**:`prompts/coach.md` 写"不评判反馈"规则 → 1 个对话练习脚本走通 → 5 个种子用户测试。

### 假设 H4(可选):同伴故事墙(Ahead 改造)

- **形态**:每周精选 3-5 个匿名用户的"我是如何……"故事,展示在"成长"页面,无评论、无点赞。
- **关键设计**:**不引导比较**(不说"看看别人多努力");**有审核与脱敏**(KE + Q 双签)。
- **风险**:可能引发"我的问题没别人严重"的反向比较;需文案克制。
- **验证**:与"用户最常回看的页面"指标挂钩。
