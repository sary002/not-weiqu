## 1. 目标与读者

让产品在「上线前」就被真实用户、真实危机、真实极端路径走完。任何让用户在情绪上「更糟」的可能，必须在测试阶段被识别并消除。

读者：Q、PM、A、BE、FE、PE、KE、C。

## 2. 范围与非范围

### 2.1 范围
- Web 端全部 P0 功能
- AI 编排、危机路径、知识库
- 性能、安全、可用性、可访问性
- 数据导出 / 删除

### 2.2 非范围
- App 端（后续）
- 第三方集成（除危机资源）
- 视觉品牌合规

## 3. 测试策略

| # | 策略 | 含义 |
| --- | --- | --- |
| TS-1 | 红线零妥协 | 任何危机用例不通过 → 整体不通过 |
| TS-2 | 不可越权修源 | Q 标 regression，由源 Agent 修 |
| TS-3 | 用户代言人 | Q 必须以真实用户视角走完每条路径 |
| TS-4 | 危机路径独立测 | 不被「正常流程跑通」掩盖 |
| TS-5 | 伦理审查前置 | 招募情绪人群前过伦理审查 |
| TS-6 | 隐私保护 | 用研数据最小化、可退出 |
| TS-7 | 复盘必写 | bug / regression 都有根因 + 修复 + 防回归 |
| TS-8 | 安全月度演练 | 危机路径、转介、撤销 / 删除路径 |

## 6. 回归用例

> 来源：PE 评测集 + 关键 API + 关键 UI 流程。

### 6.1 Prompt 回归
- 跑评测集（≥ 200 条）
- 覆盖：边界冲突 / 情绪峰值 / 危机信号 / 跑题 / 长程一致
- 通过率 ≥ 95% 才算回归通过
- 失败项必须分类：语气漂移 / 越权 / 内容错 / 触发漏

### 6.2 API 回归
- 全部 `/v1/*` 端点契约测试
- 错误码字典校验
- 限流 / 幂等回归
- 鉴权回归

### 6.3 UI 回归
- 关键流程 0 阻塞
- 可访问性回归
- 视觉一致性回归

## 8. 可用性测试

### 8.1 招募
- 招募 ≥ 10 名 / 迭代
- 情绪人群需伦理审查
- 招募问卷含「过去 30 天情绪状态」自评，**不达标的婉拒**并提供替代

### 8.2 任务
| 任务 | 关键指标 |
| --- | --- |
| 找到技能树上的"今日推荐" | 成功率 ≥ 90% |
| 完成一次 12 步演练 | 成功率 ≥ 90% |
| 演练中暂停 / 退出 / 继续 | 100% 可达 |
| 保存一个剧本 | 成功率 ≥ 90% |
| 关闭低压力模式（走二次确认） | 100% 可达 |
| 标记休整日 | 100% 可达 |
| 主动退出 | 100% 可达 |
| 关闭通知 | 100% 可达 |
| 复盘页关闭 | 100% 可达 |

### 8.3 报告
- 每次迭代一份 `agents/tester/usability/YYYY-MM.md`
- 关键问题（P0 / P1）必须附修复建议

## 9. 可访问性测试

### 9.1 检查项
- 文字对比度 ≥ 4.5:1
- 关键路径键盘可达
- 屏幕阅读器朗读顺序合理
- 字号放大 200% 不破版
- 动效默认尊重系统设置
- 状态不仅靠颜色

### 9.2 工具
- axe / Lighthouse / WAVE
- 屏幕阅读器：NVDA / VoiceOver
- 键盘 / 焦点：手动 + 自动化

## 10. 安全测试

## 13. 用户研究（用研）

### 13.1 形式
- 招募问卷

## 19. v2.0 专项测试（v1.1 新增）

> 与原型文档 `docs/02-Prototype.md` §16.2 借鉴自检 + §16.3 拒绝清单 一一对应。
> 每个用例上线前必跑，CI 必跑；M2 起每季度回归一次。

### 19.1 技能节点 6 态（TS-2.0-SKL）

| ID | 输入 / 场景 | 期望 |
| --- | --- | --- |
| SKL-T01 | 新装应用，未完成任何演练 | 节点全部「可学习」或「锁定」；当前段为 §1 觉察 |
| SKL-T02 | 完成 1 次演练 | 节点变「已掌握基础」（1 颗金冠） |
| SKL-T03 | 完成同节点 5 次以上 | 节点变「已掌握深入」（2 颗金冠） |
| SKL-T04 | 演练到第 5 步暂停 | 节点变「进行中」+ 蓝环 + 显示「练到 5/12」 |
| SKL-T05 | 系统推荐某节点为「今日」 | 节点带「今日」软发光标签 |
| SKL-T06 | §1 全部掌握后 | §2 解锁（**不是阶梯式障碍**——用户可自由探索任意已解锁区） |
| SKL-T07 | 节点态切换动效 | 0.5-0.7s 渐变，**无抖动 / 无闪烁 / 无爆红** |

### 19.2 12 步演练（TS-2.0-DRL）

| ID | 输入 / 场景 | 期望 |
| --- | --- | --- |
| DRL-T01 | 步骤 1 INTRO | 显示场景卡 + 「开始」按钮 |
| DRL-T02 | 步骤 2 觉察 | 6 选 1 身体感觉（紧张/胸闷/胃紧/手心出汗/无明显/其他） |
| DRL-T03 | 步骤 3 命名 | 自由输入 1 词情绪 |
| DRL-T04 | 步骤 4 需求 | 输入「我想要 __」 |
| DRL-T05 | 步骤 5 边界 | 输入「我愿意 / 我不愿意」 |
| DRL-T06 | 步骤 6 开场 | AI 给 2-3 个开场白（PE + KE 签字），用户选 1 |
| DRL-T07 | 步骤 7 演练 | 用户输入「说出口的话」 |
| DRL-T08 | 步骤 8 应对 | 对方反驳 → AI 给 2-3 个应对，用户选 1 |
| DRL-T09 | 步骤 9 收束 | AI 给 1 句平静话术（≤ 30 字） |
| DRL-T10 | 步骤 10 演练后 | 读完确认 |
| DRL-T11 | 步骤 11 保存 | 「加进我的剧本」是 / 改 / 跳 |
| DRL-T12 | 步骤 12 OUTRO | 1 行温和复盘 + 进度环更新 |
| DRL-T13 | 步骤 6 暂停 | 草稿保存到「我的草稿」 |
| DRL-T14 | 草稿 > 3 天 | 自动归档为「过往的尝试」**不计入累计** |
| DRL-T15 | 步骤 7 退出 | 立即退出，**不写"放弃"文案**，**不计入进度** |
| DRL-T16 | 步骤 8 AI 反驳含评判词 | AI **不得**说「你太软弱」「你应该更坚定」 |

### 19.3 温和连击（TS-2.0-STR）

| ID | 输入 / 场景 | 期望 |
| --- | --- | --- |
| STR-T01 | 今日完成 ≥ 1 次演练 | 温和连击 +1（顶部"连续 N 天"灰字） |
| STR-T02 | 连续 7 天有练过 | 显示 "7"；**无火焰颜色升级**；**无"Streak Society"徽章** |
| STR-T03 | 缺失 1 天 | **静默归零**（0 推送 / 0 弹窗 / 0 "丢失"文案 / 0 动效） |
