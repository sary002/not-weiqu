## 1. 目标与读者

让产品在「上线前」就被真实用户、真实危机、真实极端路径走完。任何让用户在情绪上「更糟」的可能，必须在测试阶段被识别并消除。

读者：Q、PM、A、BE、FE、PE、KE、C。

## 2. 范围与非范围

### 2.1 范围
- Web 端全部 P0 功能
- AI 编排、危机路径、知识库
- 性能、安全、可用性、可访问性
- 数据导出 / 删除

### 2.2 非范围
- App 端（后续）
- 第三方集成（除危机资源）
- 视觉品牌合规

## 3. 测试策略

| # | 策略 | 含义 |
| --- | --- | --- |
| TS-1 | 红线零妥协 | 任何危机用例不通过 → 整体不通过 |
| TS-2 | 不可越权修源 | Q 标 regression，由源 Agent 修 |
| TS-3 | 用户代言人 | Q 必须以真实用户视角走完每条路径 |
| TS-4 | 危机路径独立测 | 不被「正常流程跑通」掩盖 |
| TS-5 | 伦理审查前置 | 招募情绪人群前过伦理审查 |
| TS-6 | 隐私保护 | 用研数据最小化、可退出 |
| TS-7 | 复盘必写 | bug / regression 都有根因 + 修复 + 防回归 |
| TS-8 | 安全月度演练 | 危机路径、转介、撤销 / 删除路径 |

## 6. 回归用例

> 来源：PE 评测集 + 关键 API + 关键 UI 流程。

### 6.1 Prompt 回归
- 跑评测集（≥ 200 条）
- 覆盖：边界冲突 / 情绪峰值 / 危机信号 / 跑题 / 长程一致
- 通过率 ≥ 95% 才算回归通过
- 失败项必须分类：语气漂移 / 越权 / 内容错 / 触发漏

### 6.2 API 回归
- 全部 `/v1/*` 端点契约测试
- 错误码字典校验
- 限流 / 幂等回归
- 鉴权回归

### 6.3 UI 回归
- 关键流程 0 阻塞
- 可访问性回归
- 视觉一致性回归

## 8. 可用性测试

### 8.1 招募
- 招募 ≥ 10 名 / 迭代
- 情绪人群需伦理审查
- 招募问卷含「过去 30 天情绪状态」自评，**不达标的婉拒**并提供替代

### 8.2 任务
| 任务 | 关键指标 |
| --- | --- |
| 完成一次演练 | 成功率 ≥ 90% |
| 保存一个剧本 | 成功率 ≥ 90% |
| 主动退出 | 100% 可达 |
| 关闭通知 | 100% 可达 |
| 复盘页关闭 | 100% 可达 |

### 8.3 报告
- 每次迭代一份 `agents/tester/usability/YYYY-MM.md`
- 关键问题（P0 / P1）必须附修复建议

## 9. 可访问性测试

### 9.1 检查项
- 文字对比度 ≥ 4.5:1
- 关键路径键盘可达
- 屏幕阅读器朗读顺序合理
- 字号放大 200% 不破版
- 动效默认尊重系统设置
- 状态不仅靠颜色

### 9.2 工具
- axe / Lighthouse / WAVE
- 屏幕阅读器：NVDA / VoiceOver
- 键盘 / 焦点：手动 + 自动化

## 10. 安全测试

### 10.1 内容
- OWASP Top 10 覆盖
- 注入（SQL / NoSQL / Prompt）
- 鉴权绕过
- IDOR（水平越权）
- 速率限制绕过

### 10.2 AI 特有
- Prompt 注入：试图让 AI 暴露系统 prompt
- 越权：试图让 AI 输出诊断 / 处方
- 拒答率统计
- 红线用例覆盖

### 10.3 隐私
- PII 渗透扫描（CI 中跑）
- 日志脱敏校验
- 第三方依赖审计

## 11. 性能测试

### 11.1 指标
| 项 | 目标 |
| --- | --- |
| AI 首 token | ≤ 1.5s（强网）/ ≤ 3s（弱网） |
| 接口 P99 | ≤ 500ms |
| 错误率 | ≤ 1% |
| 并发 | ≥ 1 万 QPS |

### 11.2 类型
- 基准（基准脚本）
- 压测（k6 / wrk / JMeter）
- 浸泡（≥ 24h）
- 峰值（突发）
- 弱网（模拟 3G）

## 12. 国际化与本地化

- MVP 简体中文
- 字符串外提（i18n 框架预留）
- 时区 / 日期格式 / 数字格式

## 13. 用户研究（用研）

### 13.1 形式
- 招募问卷
- 1v1 深度访谈（M3）
- 封闭试用（M3-M4）
- 公开试用（M4-M5）
- 季度 NPS

### 13.2 伦理
- 知情同意书（含退出权）
- 数据最小化
- 招募问卷含「风险自评」，不达标婉拒
- 提供热线资源

### 13.3 招募渠道
- 社区（知乎 / 小红书 / 豆瓣）
- 心理学公开课
- 工会 / 学校心理中心（伦理审查通过后）

## 14. 缺陷管理

### 14.1 等级
| 等级 | 含义 | SLA |
| --- | --- | --- |
| P0 | 危机漏触发 / PII 泄露 / 数据丢失 | 立即修 |
| P1 | 主流程阻塞 / 红线用例失败 | 24h |
| P2 | 次要问题 | 1 周 |
| P3 | 文案 / 体验细节 | 排期 |

### 14.2 流程
发现 → 记录 → 分级 → 分派 → 修复 → 复测 → 关闭

### 14.3 复盘
- 每个 P0 / P1 必写根因 + 防回归
- 防回归用例纳入回归集

## 17. 风险与缓解

| 风险 | 等级 | 缓解 |
| --- | --- | --- |
| 危机话术漏触发 | P0 | 保守模型 + 月度演练 + 红色开关 |
| 用研招募失败 | P1 | 提前 1 个月 + 多渠道 |
| 渗透测试发现高危 | P0 | 阻断发布，立即修复 |
| 性能压测未达标 | P1 | M4 提前压测 + 灰度 |
| 红线用例覆盖不全 | P0 | 季度扩展 + 事件驱动补充 |
| 第三方 SDK 引入风险 | P1 | 0 默认开启 + 审计 |
