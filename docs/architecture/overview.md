# 架构总览（Overview）

> 文档状态：v0.1 草稿 · 负责人：A · 评审：PM + BE + PE

## 目标
在不引入过度复杂度的前提下，让《不委屈》能跑得稳、演得动、护得住用户的情绪与隐私。

## 形态
- **端**：Web（先）、Mobile（后）
- **后端**：单一职责的微服务（避免一开始过度拆分）
- **AI 编排**：独立服务，与业务解耦
- **数据**：用户档案（脱敏）+ 对话历史 + 进度

## 关键原则
1. **可逆 > 最优**：能回滚的选型 > 短期最优
2. **Boring Tech 默认**：AI 场景才用前沿
3. **隐私默认安全**：所有 PII 默认脱敏
4. **可观测**：每个 AI 调用可追溯、可复现

## 非功能性需求
- **可用性**：99.5% 起，逐步抬升
- **响应**：AI 首 token ≤ 1.5s
- **数据驻留**：默认中国大陆
- **审计**：所有 AI 调用保留 trace_id + prompt_hash
