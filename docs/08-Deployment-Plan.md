## 1. 目标与读者

让产品在「可灰度、可回滚、可观测、可演练」四条线上不偏。

读者：A、BE、Q、PM、C、FE。

## 2. 部署目标

| # | 目标 |
| --- | --- |
| DP-1 | 任意变更可灰度、可回滚 |
| DP-2 | 危机路径独立部署、独立压测 |
| DP-3 | 0 明文密钥出现在仓库 / 镜像 / 日志 |
| DP-4 | 灾备可演练、恢复有 SLA |
| DP-5 | 监控告警 5 分钟内可达 |

## 3. 部署形态

### 3.1 形态
- 单云单 Region（VPC 内）作为 MVP
- 多 AZ 部署
- 容器化（Docker）+ 托管容器平台
- 暂不引入 K8s 的复杂度

## 4. 环境与账号

### 4.1 环境
| 环境 | 用途 | 数据 |
| --- | --- | --- |
| dev | 工程师本地 | 合成数据 |
| ci | 流水线集成 | 合成数据 |
| staging | 上线前回归 | 真实数据脱敏 |
| canary | 灰度（按租户） | 真实数据 |
| prod | 生产 | 真实数据 |

### 4.2 账号
- 服务账号最小权限
- 不共享个人账号
- 双人复核（关键操作）

## 5. 流水线

### 5.1 阶段
```
代码 → Lint → 单测 → 集成 → 镜像构建 → 漏洞扫描 → 契约测试
   → 部署 staging → E2E + 红线用例 → 人工审批
   → 部署 canary（按租户灰度）→ 监控观察
   → 全量 prod
```

### 5.2 阻断条件（任一触发即停）
- 单测失败
- 集成失败
- 漏洞扫描高危
- 契约测试失败
- 红线用例失败
- 监控告警未就绪

### 5.3 工具
| 阶段 | 工具 |
| --- | --- |
| Lint | ESLint / golangci-lint |
| 单测 | go test / pytest |
| 集成 | 自研 runner |
| 镜像 | Docker + 托管镜像仓库 |
| 漏洞 | Trivy / Snyk |
| 契约 | Pact / Spectral |
| 部署 | 托管容器平台 / Argo CD（视选型） |
| 监控 | Prometheus + Grafana + Loki + Jaeger |

## 6. 镜像与制品

- 基础镜像：官方 distroless / alpine
- 镜像签名（cosign）
- 制品版本 = Git SHA + 时间戳
- 不在镜像中写入密钥

## 7. 配置与密钥

### 7.1 配置
- 配置中心统一管理
- 12-factor：环境变量 + 配置中心
- 变更走审批 + 留痕

### 7.2 密钥
- KMS 托管
- 动态凭证（服务间）
- 轮转周期：90 天
- 不在代码 / 镜像 / 日志中

## 8. 数据库与迁移

- 迁移工具：golang-migrate（按语言定）
- DDL 必须有 up / down
- 大表变更：影子表 + 切流
- 备份：每日全量 + WAL 归档
- 异地灾备：同城不同 AZ
- 季度演练恢复

## 9. 灰度与回滚

### 9.1 灰度策略
- 按租户 ID 灰度
- 比例：1% → 5% → 20% → 50% → 100%
- 每个阶段观察窗口 ≥ 30 分钟
- 关键指标异常即停

### 9.2 回滚
- 任意阶段可一键回滚
- 回滚不丢数据
- 回滚后自动出 P1 工单

### 9.3 灰度开关
- Feature Flag（自建或 LaunchDarkly）
- 危险开关：「关闭 AI 输出」「关闭演练」「关闭危机」

## 10. 监控与告警

### 10.1 三大支柱
- Metrics：Prometheus
- Logs：Loki / ELK
- Traces：OpenTelemetry + Jaeger

### 10.2 关键告警
| 指标 | 阈值 | 等级 |
| --- | --- | --- |
| 错误率 | > 1% 持续 5min | P1 |
| AI 首 token | > 3s 持续 5min | P1 |
| 危机路径调用失败 | > 0 | P0 |
| 数据删除 SLA | > 24h | P1 |
| PII 在日志 | 1 次 | P0 |
| 备份失败 | 1 次 | P1 |
| 磁盘使用 | > 80% | P2 |
| 复制延迟 | > 5s 持续 5min | P1 |

### 10.3 通知
- 飞书 / 钉钉 / 企微（按团队选）
- P0 立即电话
- P1 立即 IM
- P2 工作时间 IM

## 11. 灾备与演练

| 项 | 目标 |
| --- | --- |
| RPO | ≤ 1h |
| RTO | ≤ 30min |
| 演练 | 季度 1 次 |
| 演练结果 | 写入 `docs/decisions/adr-NNN-dr-<date>.md` |

## 12. 安全与合规

### 12.1 边缘
- WAF、Bot 防护、CC 防护
- TLS 1.2+、HSTS
- 限速：API、登录、注册、数据请求

### 12.2 数据
- 静态加密（AES-256）
- 传输加密
- KMS 密钥轮转
- 备份加密

### 12.3 合规
- 隐私政策 + 用户协议
- Cookie 横幅（如启用）
- 数据驻留：中国大陆
- 跨境：不主动跨境

## 13. 危机路径独立部署

### 13.1 独立命名空间
- 独立部署单元
- 独立数据库（强一致）
- 独立监控 / 告警
- 独立 CI / CD 流水线

### 13.2 不被降级覆盖
- 即便上游 LLM 全挂，危机话术本地兜底必须可用
- 监控告警：危机路径调用失败 → 立即 P0
- 季度独立压测

### 13.3 演练
- 月度红队演练（Q 主持）
- 演练结果必写报告

## 14. 风险与缓解

| 风险 | 等级 | 缓解 |
| --- | --- | --- |
| 单 Region 故障 | P0 | 多 AZ + 灾备演练 |
| 镜像供应链攻击 | P0 | 镜像签名 + 漏洞扫描 |
| 密钥泄露 | P0 | KMS + 轮转 + 监控 |
| 数据库误操作 | P0 | 软删除 + 备份 + 二次确认 |
| 灰度策略失效 | P1 | 双灰度（租户 + 比例） |
| 监控盲区 | P0 | 关键指标强制就绪 |
| 危机路径失联 | P0 | 独立部署 + 本地兜底 |
| 第三方依赖故障 | P1 | 关键依赖 ≥ 2 家 |

## 15. 附录

### 15.1 发布 Runbook（节选）
1. 确认 PR 已合并主干
2. 确认 staging 全绿
3. 发起 canary 部署
4. 监控观察 30 分钟
5. 比例提升
6. 全量
7. 观察 24h
8. 写发布纪要

### 15.2 回滚 Runbook（节选）
1. 立即触发危险开关
2. 切回上一个稳定版本
3. 通知值班
4. 写 P1 工单
5. 根因分析
6. 防回归用例入库
