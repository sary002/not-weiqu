## 3. 阶段定义

### 3.1 M0 治理冻结（2026-06）
- **目标**：把项目治理体系搭起来
- **产出**：8 Agent、4 规则、9 设计文档、CLAUDE.md
- **质量门**：G1（详见 `06-Development-Plan.md`）
- **退出标准**：L1 评审通过 + 9 份文档齐 + 红线用例已建立

### 3.2 M1 设计冻结（2026-07）
- **目标**：所有设计文档 v1.0 通过 L2 评审
- **产出**：PRD、原型、架构、数据库、API、知识块 v0.5、人格 v1.0、评测集 v0.5、测试方案、部署方案
- **质量门**：G2
- **退出标准**：L2 评审全过 + 首批 ADR 接受

### 3.3 M2 PoC（2026-08 · v2.0 形态）
- **目标**：1 微技能 + 12 步演练 + 温和连击 + 低压力模式 + 危机兜底 端到端跑通
- **产出**：
  - **技能树主页**（5 段骨架 + 节点 6 态）
  - **12 步演练端到端**（L3 表达 - 拒绝同事甩活 1 个微技能完整跑通）
  - **今日三件小事**（1 必做 + 2 选做）
  - **温和连击**（缺失静默归零 + 每周休整日）
  - **低压力模式**（默认 ON + 顶部常驻开关）
  - 危机话术 + 转介页
  - 评测集 v0.5（含温和连击 / 低压力话术维度）
  - 部署流水线 v1
- **质量门**：G3
- **退出标准**：
  - 1 个微技能 12 步端到端走通
  - 危机话术演练通过
  - **拒绝清单自检通过**（代码库无 heart / gem / leaderboard 字段）
  - 推送默认全关验证
  - 文案不含催促型话术验证

### 3.4 M3 内测（2026-10 · v2.0 形态）
- **目标**：100 名真实用户封闭试用
- **产出**：
  - 全部 P0 模块（5 段技能树全部微技能 + 知识小卡 + 完整 4 Tab）
  - 知识块 200+（**对齐 5 段微技能库**）
  - 监控告警（**新增温和连击异常 / 低压力模式冲突监控**）
  - 用研报告
  - **拒绝清单回归**（确保未引入心数 / 宝石 / 排行榜 / 打卡天数 / 付费修复）
- **质量门**：G4
- **退出标准**：封闭用户用研通过 + 监控告警就绪 + 拒绝清单 100% 通过

### 3.5 M4 公测（2026-12）
- **目标**：1k 公开用户灰度
- **产出**：性能压测报告、安全渗透报告、假设验证结论
- **质量门**：G5
- **退出标准**：渗透通过 + SLA 达标

### 3.6 M5 正式（2027-03）
- **目标**：正式发布
- **产出**：SLA 化、危机路径独立压测、发布纪要
- **质量门**：G6（每阶段都过）
- **退出标准**：发布纪要 + 复盘 + 假设回流

## 6. 能力地图与上线顺序

### 6.1 能力地图
```
                    ┌──────────────────┐
                    │      安全兜底     │  ← M2 上线
                    └────────┬─────────┘
                             │
              ┌──────────────┼──────────────┐
              ▼              ▼              ▼
        ┌─────────┐    ┌─────────┐    ┌─────────┐
        │ 觉察 L1 │    │ 命名 L2 │    │ 表达 L3 │  ← M2~M3
        └────┬────┘    └────┬────┘    └────┬────┘
             │              │              │
             └──────┬───────┴──────┬───────┘
                    ▼              ▼
              ┌──────────┐  ┌──────────┐
              │ 兜底 L4  │  │ 巩固 L5  │  ← M3~M4
              └──────────┘  └──────────┘
```

### 6.2 上线顺序（v2.0 形态）
1. **M2 必上**：
   - 危机兜底（L4，独立验证）
   - **技能树主页**（5 段骨架 + 节点 6 态）
   - **12 步演练**（L3 表达 - 拒绝同事甩活 1 个微技能）
   - **温和连击**（缺失静默归零 + 每周休整日）
   - **低压力模式**（默认 ON）
2. **M3 必上**：
   - 全部 5 段技能树（§1 觉察 / §2 命名 / §3 表达 / §4 兜底 / §5 巩固）
   - L1 觉察 / L2 命名 / L3 表达 微技能
   - **今日三件小事**（1 必做 + 2 选做）
   - 知识小卡
3. **M3 末**：L4 兜底完整话术（被回绝后自我安抚 / 应对愧疚感）
4. **M4 上**：L5 巩固（30 日回顾 / 整理我的剧本集）

> 「危机兜底」必须最先验证，因为它是产品的命脉。
> 「低压力模式」默认 ON 是产品的伦理底线，比功能完整更重要。

### 10.2 名词
- KM：Key Milestone
- L1~L5：边界五层模型
- PoC：Proof of Concept
- SLA：Service Level Agreement
- NPS：Net Promoter Score
- **微技能（Skill）**：技能树上的一个节点，对应 5–7 个演练
- **演练（Drill）**：12 步固定流程
- **温和连击（Gentle Streak）**：连续 N 天有"练过"；缺失静默归零
- **低压力模式（Low Pressure Mode）**：默认 ON；静默所有推送 / 红点 / 动效
- **拒绝清单（Reject List）**：v2.0 明确不引入的元素（心数 / 宝石 / 排行榜 / 打卡天数 / 付费修复 / 催促型推送）
