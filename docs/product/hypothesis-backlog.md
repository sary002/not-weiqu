# 假设清单（Hypothesis Backlog）

> 文档状态：v0.1 · 负责人：PM
> 我们不接受「我觉得用户需要 X」式的需求。每个需求都对应一个可证伪假设。

## 假设模板
```
H-<NNN>  若 <动作>，则 <可观察结果>，因为 <机制>。
        验证方式：<指标 / 试验>
        失败回退：<下一步假设或放弃>
```

## 待验证

### H-001 场景演练比纯科普更能促进行动
- 若：用户完成 1 次「拒绝同事甩活」的角色扮演
- 则：3 日内至少 1 次真实场景尝试
- 因为：演练降低「第一次说不」的心理成本
- 验证：A/B 演练组 vs 纯阅读组
- 失败回退：增加「现实场景自定义」入口

### H-002 深夜 / 周末独处时段是用户的情绪低谷高峰
- 若：在 22:00-01:00 提供轻量入口
- 则：情绪记录密度提升 ≥ 30%
- 验证：分时段埋点
- 失败回退：重新做时段调研

### H-003 「不说教」比「金句」更能带来复访
- 若：每条提示词开头都有共情句
- 则：30 日复访率 ≥ 35%
- 验证：NPS + 复访率
- 失败回退：提示词评测集改造
