# ADR-005: 灵魂设计 v2（让用户感到"被看见"）

- **状态**：Proposed
- **日期**：2026-06-17
- **决策者**：PM（主）+ PE（联合）+ 用户（产品方向）
- **影响模块**：今日页 / 自由对话 / 12 步演练 / 危机兜底 / 设置 / 早晚问候
- **触发**：用户对"今日练习没有灵魂"的反馈

---

## 背景

### 用户反馈（2026-06-17）

> "目前的今日练习没有灵魂"

### 当前问题

| 当前形态 | 缺什么 |
|---|---|
| 直接进任务列表："今日 1/3 已完成" | 没有"准备"的过程 |
| AI 中性回复："我理解你的感受" | 没有"人"的温度 |
| 完成 = 节点翻转 + 进度圈变色 | 没有"回响" |
| 启动后界面都一样 | 没有"时间"感知 |
| 统一对待所有用户 | 没有"个体"感知 |

### 关键约束

- CLAUDE.md §1：不评判、不催促、不替代专业心理咨询
- CLAUDE.md §5 红线 1-8：不用评判工具 / 不用 🔥 / 不用打卡天数 / 不用催促
- rules/safety.md S-3：危机路径独立 + 不依赖 LLM
- rules/safety.md S-4：语气红线（不评判 / 不催）
- 已有 v0.1 资产：`agents/prompt-engineer/persona.md`（温姐 / 智哥 / 松松）+ `prompts/coach-not-judging.md`（不评判守则）

### 设计原则（CLAUDE.md §1 + §5 红线）

> **灵魂 ≠ 热闹 ≠ 动画 ≠ 🎉彩带。灵魂 = 让用户感到"被看见"。**

---

## 决策

我们选择**完整实施灵魂设计 v2**，4 个改进方向按优先级：

### 🔴 P0 · 进入练习前先"看见"我（情境卡）

**现状**：直接进任务列表。

**改进**：进入今日页时先弹一个 3~5 秒的**情境卡**：

```
┌──────────────────────────────────┐
│                                  │
│  今天感觉怎么样？                  │
│                                  │
│  🌧  低落                        │
│  🌤  一般                        │
│  ☀️  还不错                      │
│  ✨  平静                        │
│                                  │
│  （选完进入今日）                  │
└──────────────────────────────────┘
```

**AI 回应（基于情绪 + 历史）**：

| 情绪 | AI 回应 |
|---|---|
| 低落 | "嗯，今天有点累。进来坐坐就好，不一定非要做练习。" |
| 一般 | "今天是这样的日子。我们走一步看一步？" |
| 还不错 | "听到这个我也开心了一点点。今天想练什么？" |
| 平静 | "嗯，平静也是一种力量。今天想做什么？" |

**为什么不评判原则不冲突**：让用户**自己命名情绪**，AI 不评判不解读，**只回响**。

### 🔴 P0 · AI 人格化（3 人格可选）

**现状**：AI 是中性的"助手"。

**改进**：3 个可选人格（已有 v0.1 在 `agents/prompt-engineer/persona.md`）：

| 人格 | 适合 | 语气特征 |
|---|---|---|
| **温姐** | 情绪低落时 | 温柔 / 共情 / "嗯嗯" / "我在听" / "你愿意继续说吗" |
| **智哥** | 焦虑时 | 理性 / 结构化 / "我们一步步来" / "先厘清事实" |
| **松松** | 压力大时 | 轻松 / 幽默 / "诶？这事儿有意思" / "笑一下也行" |

**用户可自选 + 系统根据情境推荐**：

- 进入今日页时检测最近 3 次情绪选择：连续 2 次"低落" → 推荐温姐
- 演练 12 步前 3 步节奏很快 → 推荐智哥（结构化）
- 周末 + 多次"平静" → 推荐松松（轻松）

**人格不评判、不催促，只是表达方式不同**。

### 🟡 P1 · 完成后有"回响"

**现状**：完成 = 节点翻转 + 进度圈变色。无反馈。

**改进**：
- 完成时 AI 给一句**具体的、个性化的反馈**（基于这次输入）：
  - ❌ "你真棒"（评判）
  - ✅ "你刚才说『今晚我有安排了』—— 这句话里能看到你已经在保护自己的节奏"（**回响**）
- 每周日晚有「本周回顾」：AI 用 3 句话总结这周 7 天（不是数据，是感受）

**回响模板**（PE 撰写示例）：

```
你刚才说：[用户原话引用]
这句话里能看到：[观察 / 共情]
```

### 🟡 P1 · 早晨 + 晚安问候

**现状**：没有时间感知。

**改进**：
- **早晨**（首次打开）：根据上次结束时间 + 节日给一句问候
  - "早，又是新的一天。"
  - "周一的早晨。我们慢慢来。"
  - "周五了。你这周已经练过 3 次。"
- **晚间 22:00 后**：自动切到「夜话模式」—— UI 变暗 + 推荐变成"如果你今晚不想练，可以写几句话"
- **深夜 0:00+ 不打扰**（推送默认 0 条）

---

## 灵魂的 3 条铁律

> **灵魂 ≠ 热闹。灵魂 = 安静地被看见。**

1. **不打扰**：用户不打开时，绝不主动打扰（推送默认 0 条）
2. **不评判**：完成 ≠ 胜利，缺勤 ≠ 失败
3. **不催促**：永远给"今天就到这里"的退出路径（已在 today/page.tsx 实装）

---

## 备选

### 选项 A：完整实施（4 个方向）

- 实施全部 4 个方向
- 工时：~16 人日
- 价值：完整灵魂体验

### 选项 B：仅实施 P0（情境卡 + AI 人格化）

- 实施 2 个 P0 方向
- 工时：~8 人日
- 价值：核心灵魂体验

### 选项 C：仅实施 P0 情境卡

- 实施 1 个 P0 方向
- 工时：~3 人日
- 价值：最小的"被看见"感

### 选项 D：不做事

- 现状维持
- 工时：0
- 价值：用户反馈未解决

---

## 决策（详细）

我们选择 **选项 B：仅实施 P0（情境卡 + AI 人格化）**，理由：

1. **P0 是核心**：情境卡 + AI 人格化是"被看见"的最直接体现
2. **P1 可后续迭代**：完成回响 + 早晚问候可在 M3 真实用户阶段补
3. **工时可控**：~8 人日，可在 M2 PoC 周期内完成
4. **风险最低**：P0 改动都是新增功能，不影响现有 13 份设计稿

---

## 后果

### 得到

- ✅ 用户进入应用时感到"被看见"（情境卡）
- ✅ AI 有"人"的温度（人格化）
- ✅ 用户与 AI 的关系从"工具"变成"陪伴"
- ✅ 与 CLAUDE.md §1「不评判」原则更贴合（AI 只回响不评判）

### 失去

- ❌ 实施工时 ~8 人日（M2 PoC 周期内可控）
- ❌ 需要 PE 撰写大量守则内容（人格 / 情境回应模板）

### 风险

| ID | 风险 | 等级 | 缓解 |
| --- | --- | --- | --- |
| R-1 | 情境卡弹窗打断用户流程 | P1 | 可跳过（"下次再说"按钮）+ 7 天内不重复弹 |
| R-2 | AI 人格推荐让用户感觉被"算" | P1 | 首次默认温姐（最不侵入）；切换无门槛 |
| R-3 | 人格之间切换让用户困惑 | P2 | UI 明确展示当前人格 + "换一种试试"按钮 |
| R-4 | 情境卡变成"打卡"（用户觉得被追踪） | P1 | 仅首次进入时弹；后续完全跳过 |
| R-5 | 早晨问候变成"推送" | P1 | 仅在用户主动打开 app 时显示；不推送 |

### 回滚条件

满足任一即回滚或调整：

- 用户对情境卡 / 人格化反馈负面（≥ 30% 用户表示"不喜欢"）
- 灵魂设计消耗过多 LLM token（单次 > 500 tokens）
- 与现有危机路径（ADR-002 紧急联系人 / T-09 设计稿）冲突

---

## 安全与合规自检

- [x] **是否涉及 PII？**
  - 否（情境卡是用户自选情绪，不收集额外信息）
  - 但 LLM 读取用户最近输入生成回应——已确认不输出 PII
- [x] **是否影响用户情绪安全？**
  - **是**（灵魂设计直接处理用户情绪）
  - 缓解：
    - 情境卡让用户**自己命名情绪**（不评判不解读）
    - AI 人格只回响不评判（"嗯嗯" / "我在听"）
    - 危机关键词仍由 L0 规则层检测（不依赖灵魂设计）
- [x] **是否需要 Q 红线用例？**
  - **是**：
    - 情境卡跳过路径完整（"下次再说"按钮可达）
    - 人格切换不影响危机路径触发
    - 早晨问候不推送（仅首次打开时显示）
    - 7 天内情境卡不重复弹
    - 人格推荐可手动覆盖（用户可自选）

## 数据契约

### 情境卡选择（M2 存 localStorage）

```typescript
// localStorage key: nw-mood-checkin-v1
type MoodEntry = {
  date: string;           // YYYY-MM-DD
  mood: 'low' | 'normal' | 'good' | 'calm';
  chosenPersona?: 'wen' | 'zhi' | 'song';
};

type MoodCheckinStorage = {
  version: 1;
  entries: MoodEntry[];   // 最近 30 天
};
```

### 用户人格偏好（M2 存 localStorage）

```typescript
// localStorage key: nw-persona-pref-v1
type PersonaPref = {
  version: 1;
  current: 'wen' | 'zhi' | 'song';   // 当前人格
  autoRecommend: boolean;               // 是否启用自动推荐
};
```

### M3 末迁 Supabase

```sql
-- profiles.mood_checkin JSONB
ALTER TABLE profiles
  ADD COLUMN mood_checkin JSONB DEFAULT '{}'::jsonb;

-- profiles.persona_pref TEXT
ALTER TABLE profiles
  ADD COLUMN persona_pref TEXT DEFAULT 'wen';

-- RLS：用户只能读写自己的
CREATE POLICY "Users manage own mood/persona"
  ON profiles FOR ALL
  USING (auth.uid() = id);
```
