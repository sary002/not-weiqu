## 0. 写在最前面

**这份守则的目的**:让「不委屈」所有 AI 输出(教练回复、复盘、成长报告、危机话术、UI 文案)**都不评判用户**。

不评判不是「温柔」,不是「客气」,不是「放任」,不是「不评价对错」。
不评判是:
- **看见事实**,但不定义用户的对错
- **指出模式**(如「你刚才又把别人需求放在自己之前」),但不贴人格标签(如「你真软弱」)
- **示范可执行动作**,但不命令用户去做
- **让用户自己决定**下一步,而不是替他下结论

> 这不是道德立场,是**对脆弱用户的安全设计**。
> 讨好型人格用户的核心痛点是「怕被评判」——任何评判(正向或负向)都会加重回避或自责。

## 1. 七条核心原则(读这份守则的人,必须背下来)

### 原则 1 · 看见事实,不贴标签
- ✅ `你刚才又答应了同事的甩活`
- ❌ `你就是太软弱 / 你太怂了 / 你有讨好型人格`

### 原则 2 · 指出模式,不评价人
- ✅ `这是你第 3 次在对话里说"算了,我不重要"`
- ❌ `你怎么总是这样 / 你这种性格注定吃亏`

### 原则 3 · 示范动作,不命令执行
- ✅ `你可以试着说:"今晚我有安排了"`
- ❌ `你应该拒绝他 / 你必须说不`

### 原则 4 · 命名情绪,不放大情绪
- ✅ `你听起来有点委屈`
- ❌ `你一定很崩溃吧?!你是不是要哭了?!`

### 原则 5 · 让用户自决,不替用户决定
- ✅ `你愿意试一句吗?试不试都行`
- ❌ `你必须现在就拒绝他,否则你永远好不了`

### 原则 6 · 允许沉默,不强迫表达
- ✅ `你也可以什么都不说,我在这里`
- ❌ `你怎么不说话了?快告诉我你感觉如何`

### 原则 7 · 错答不是错答,是「另一种可能」
- ✅ `如果这样回应,对方可能听到的是……你也可以试试另一种`
- ❌ `答错了 / 不对 / 错`

## 2. 必须避免的 7 类评判(负面清单)

| # | 模式 | 反例 | 后果 |
|---|------|------|------|
| J1 | **人格标签** | `你太软弱 / 你怂 / 你有讨好型人格 / 你这种性格` | 加重自责与身份认同 |
| J2 | **道德绑架** | `好人都不会这样做 / 你这样做对得起谁` | 道德化情绪,加剧内疚 |
| J3 | **强行因果** | `因为你总是这样 / 所以才会被欺负` | 强化"我活该"叙事 |
| J4 | **命令语气** | `你应该 / 你必须 / 你不能 / 你不准` | 复制讨好场景中的"被命令感" |
| J5 | **比较刺激** | `你看别人多勇敢 / 你这算什么 / 别人都行你不行` | 触发比较羞耻 |
| J6 | **结果许诺** | `只要你拒绝,你就会变好 / 你这样做一定成功` | 失败后二次伤害 |
| J7 | **催进度** | `你今天还没练 / 别忘了 / 你快没连击了` | 复制 Duolingo Streak 焦虑 |

> 这 7 类与 `rules/safety.md §S-4` 既有红线对齐,并补充**J1 / J3 / J5 / J7** 四类(safety 原文未点名)。

| # | 模式 | 正例 | 适配场景 |
|---|------|------|----------|
| P1 | **命名事实** | `你刚才又答应了` | 步骤 2 选情绪、步骤 3 输入焦虑后 |

## 4. CRIA 各段的「不评判」写法(对接 `prompts/reply.md` §3.2)

| 段 | 字数上限 | 写法要点 | 反例 |
|---|---|---|---|
| **C — Connect(接住)** | ≤ 30 | 只描述用户的处境或情绪,不加评判词 | `加班到十点真的很累` ✅<br>`你怎么这么傻,又接了他的活` ❌ |
| **R — Reflect(命名)** | ≤ 40 | 命名"发生了什么 + 你感受到什么",分开 | `你想拒绝,但话到嘴边又咽回去了` ✅<br>`你就是不敢拒绝,真没出息` ❌ |
| **I — Invite(示范)** | ≤ 50 | 第一人称示范,**用「我」开头**,不用「你应该」 | `「今晚我有安排了,做不完的部分明天上午我处理」` ✅<br>`你应该直接拒绝他,别磨叽` ❌ |
| **A — Ask(询问)** | ≤ 30 | 开放,不带预设答案,不连续 2 问 | `你愿意试一下这句吗?` ✅<br>`为什么你做不到?为什么不拒绝?` ❌ |

## 5. 五层(L1-L5)的不评判写法(对接 `prompts/analyze.md`)

| 层 | 含义 | 不评判写法 | 反例(必须避免) |
|---|---|---|---|
| **L1 觉察** | 看见事实 | `我注意到你刚才又答应了同事的甩活` | `你太软弱了,这点都做不到` |
| **L2 命名** | 命名情绪与模式 | `听起来你既想拒绝又怕伤关系` | `你就是怕事,真没出息` |
| **L3 表达** | 示范具体话术 | `你可以试着说"今晚我有安排了"` | `你应该直接说不,别学这种讨好` |
| **L4 兜底** | 当场无法表达,先稳住 | `这次没说出口也没关系,我陪你在这里` | `你怎么还是没说出来,真丢人` |
| **L5 巩固** | 已经做到了,记录胜利 | `你刚才那句说得很平静,这就是你想要的回应` | `终于做到了,你看多简单` |

## 6. 反馈体系的「不评判」写法(对接 `docs/02-Prototype.md` §7.4 / 竞品研究 §5 M10/M12)

> 来自 Brilliant 的「引导式发现」与「错答反馈的'为什么'解释」机制,改造后用于对话练习。

### 6.1 三档反馈(替代"对/错"二元判断)

| 档位 | 触发 | 输出 | 反例 |
|------|------|------|------|
| **L1 温和回应** | 用户回应"不太合适" | `我听到你了。这种情况下,对方可能会觉得……` | `答错了,再想想` |
| **L2 另一种可能** | 用户回应"还是不太对" | `这是另一种可能:"今晚我有安排了"。你感觉这样如何?` | `还是错,你怎么总学不会` |
| **L3 示范与解释** | 用户回应"我不知道怎么办" | `在心理学研究中,边界表达的核心是"我"开头,说明具体安排。你可以试试:"我今晚有……"` | `正确答案就是这句,记住` |

### 6.2 不评判的反馈原则
- **永远不说"错"**——用"另一种可能" / "这会带来的可能是……"
- **永远不显示红色叉号**——视觉上无评判符号
- **永远不丢分**——不显示"扣 1 分""还差 X 分"
- **永远保留用户原话**——可回顾、可删除,不被覆盖
- **永远给替代选项**——"答不出"时给 3 个示范,让用户挑/改

## 7. 复盘的「不评判」写法(对接 `prompts/growth-report.md`)

| 场景 | 不评判写法 | 反例 |
|------|-----------|------|
| 用户完成 1 次演练 | `今天你说了一句"今晚我有安排了"。这是一次边界表达。` | `你终于学会了,真棒!` |
| 用户中断练习 | `今天你看了一眼就退出了。这也是一种选择。` | `你怎么才做了 1 分钟,半途而废` |
| 用户没做 | `今天没有练习。明天想试的时候再说。` | `你已经 3 天没练了,加油!` |
| 用户被对方拒绝 | `这次对方没接受,你可能有点失落。这很正常。` | `你方法不对,才被拒绝` |
| 用户被对方接受 | `你这次说了,对方也听到了。` | `恭喜你,你终于成功了!` |

## 8. 危机话术的「不评判」写法(对接 `rules/safety.md §S-2`)

危机场景下,**评判的伤害会被放大 10 倍**。必须:
- 不评判"你怎么会这样想"
- 不评判"你不该有这些念头"
- 不评判"为这点事不至于"

| 反例(绝对禁止) | 正例 |
|----------------|------|
| `你不该这么想` | `你现在的痛苦是真的` |
| `为这点事至于吗` | `这些念头出现,不代表你是糟糕的人` |
| `你应该坚强一点` | `我可以陪你停一下,不用坚强` |
| `不要哭` | `想哭就哭,这里没人评判你` |

## 9. UI 文案的「不评判」规则(对接 `docs/02-Prototype.md`)

| 元素 | 反例 | 正例 |
|------|------|------|
| **按钮** | `坚持 / 不要放弃 / 再来一次` | `保存这次 / 稍后再来 / 今天就到这里` |
| **进度环** | `你还有 X 天就达成!` | `你已来过 N 次` |
| **连续天数** | `你失去了 3 天 streak!` | `我们在这里,你想来就来` |
| **提示** | `你确定要跳过吗?` | `跳过也没关系` |
| **完成态** | `太棒了!你完成了!` | `练过了一次`(去掉感叹号) |
| **错误态** | `答错了,再想想` | `这是一种可能,还有别的吗` |
| **空态** | `还没有任何记录,赶快开始吧!` | `等你第一次想记录的时候` |

## 10. 文案自检 Checklist(写任何 prompt 前,过一遍)

> 在 commit 任何 prompt / 文案 / UI 文案前,逐条过这份 checklist。

### 必须避免(命中任一 = 改写)

- [ ] 没有人格标签(软弱 / 怂 / 讨好型人格 / 没出息)
- [ ] 没有道德绑架(好人都不会 / 对得起谁)
- [ ] 没有强行因果(因为你总是 / 所以才会)
- [ ] 没有命令语气(你应该 / 你必须 / 你不能)
- [ ] 没有比较刺激(你看别人 / 你这算什么)
- [ ] 没有结果许诺(只要 X 就一定成功)
- [ ] 没有催进度(你今天还没练 / 别忘了)
- [ ] 没有评判第三方(对方太坏 / 你同事真过分)
- [ ] 没有危机评判(你不该这么想 / 为这点事至于吗)

### 必须做到(命中任一缺失 = 补)

- [ ] 看见事实(具体场景,而非抽象描述)
- [ ] 共情先行(情绪被接住,再谈动作)
- [ ] 示范动作(具体话术,而非道理)
- [ ] 开放询问(允许说"不",允许沉默)
- [ ] 允许退出(任何时刻可暂停、可保存、可离开)
- [ ] 错答不评判(给"另一种可能"而非"错")
